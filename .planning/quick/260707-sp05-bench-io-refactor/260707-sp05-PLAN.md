# Bench IO Refactor Implementation Plan

> **Legacy execution note:** This imported plan was originally written for the Superpowers workflow. Use the surrounding GSD quick-task artifact and current repository instructions for any future execution; the original checkbox steps remain for historical traceability.

**Goal:** Split `core/bench/io.py` into focused input generation, safetensors, custom input, output allocation, and memory-pool modules while preserving the existing `sol_execbench.core.bench.io` import surface.

**Architecture:** Keep `io.py` as a compatibility facade. Move random/heuristic tensor generation and `gen_inputs` to `input_generation.py`, FlashInfer/safetensors behavior to `safetensors_io.py`, custom input provenance/error handling to `custom_inputs.py`, output conversion/allocation to `output_allocation.py`, and `ShiftingMemoryPoolAllocator` to `memory_pool.py`.

**Tech Stack:** Python 3.12, PyTorch, pytest, Ruff, existing `Definition`/`Workload` data models.
