# Dataset Solutions Refactor Implementation Plan

> **Legacy execution note:** This imported plan was originally written for the Superpowers workflow. Use the surrounding GSD quick-task artifact and current repository instructions for any future execution; the original checkbox steps remain for historical traceability.

**Goal:** Extract dataset solution construction and source sanitizing helpers from `runner.py` into `solutions.py`, updating internal callers directly without runner compatibility re-exports.

**Architecture:** `solutions.py` owns token-aware source sanitizing, destination-passing-style inference, and conversion from definitions/local solution files to Solution dictionaries. `runner.py` keeps timing evidence, summary reporting, and derived report orchestration. Scripts and tests import solution helpers from `solutions.py` directly.

**Tech Stack:** Python 3.12, pytest, Ruff, existing SOL ExecBench dataset scripts.

---

## File Structure

- Create `src/sol_execbench/core/dataset/solutions.py`: focused solution-building helpers.
- Modify `src/sol_execbench/core/dataset/runner.py`: remove solution-building helpers and imports only needed by them.
- Modify `scripts/run_dataset.py`: import solution helpers from `solutions.py` instead of `runner.py`.
- Modify `scripts/internal/rdna4/run_rdna4_profiler_timing_batch.py`: import `build_reference_solution` from `solutions.py`.
- Modify `tests/sol_execbench/test_dataset_runner.py`: point solution helper tests at `solutions.py`; keep non-solution tests on their current modules.
