# Phase 160 Plan

## Goal

Generalize Phase 159 into a conservative closure workflow for remaining
`partial_profiler_backed` and `profiler_blocked` RDNA4 timing targets.

## Tasks

1. Add a remaining-target audit script.
   - Read the current 235-problem profiler timing coverage.
   - Select `partial_profiler_backed` and `profiler_blocked` targets by
     default.
   - Report problem id, workload count, evidence path, failure reason,
     profiler rows, trace status counts, and recommended action.

2. Emit machine-readable closure artifacts.
   - JSON report with target rows and totals.
   - Plain problem-id list suitable for `--only-problem` loops.
   - Markdown summary for review.

3. Keep targeting conservative.
   - Do not include `timing_fallback` unless explicitly requested later.
   - Do not include readiness-blocked targets.
   - Do not count partial or blocked evidence as closure.

4. Add CPU-safe tests.
   - Verify partial/blocked target selection.
   - Verify fallback/readiness-blocked exclusion by default.
   - Verify recommended action classification.

5. Run the audit on current RDNA4 coverage.
   - Generate `out/rdna4-profiler-sharded-closure-audit-20260608`.
   - Use the audit to choose the next small real sharded closure candidate.
