# Phase 183 Plan: Freshness Identity and Artifact References

## Implementation Plan

1. Extend `src/sol_execbench/core/bench/agent_feedback.py` with strict models
   for sidecar identity, artifact citations, and freshness validation status.
2. Add checksum-backed artifact citation helpers that store compact file names
   rather than unstable absolute paths.
3. Add a validator helper that classifies expected identity mismatches as
   stale and insufficient expected identity as unknown.
4. Update CLI wiring to pass the trace path and written sidecar paths into the
   agent-feedback builder after canonical trace emission.
5. Extend tests for identity fields, artifact checksums, stale/current/unknown
   freshness classification, and CLI sidecar citation output.
