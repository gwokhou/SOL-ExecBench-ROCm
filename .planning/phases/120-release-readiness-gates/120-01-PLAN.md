# Phase 120 Plan: Release Readiness Gates

## Objective

Add a prerelease readiness gate that validates the Phase 119 artifact bundle and claim-boundary state before maintainers publish a prerelease.

## Scope

- Implement a CPU-safe readiness script under `scripts/`.
- Validate required bundle files, checksums, manifest schema, authority classes, claim-boundary booleans, and known-gap statuses.
- Generate JSON and Markdown reports.
- Add focused tests and documentation.
- Update Phase 120 requirements and planning state after verification.

## Tasks

1. Implement `scripts/check_prerelease_readiness.py`.
   - Accept `--bundle-dir`, `--output-dir`, and optional `--skip-doc-claim-checks`.
   - Load `prerelease_artifact_bundle.json` and `SHA256SUMS`.
   - Fail missing required files, checksum mismatches, unknown authority classes, forbidden truthy claim fields, and invalid known-gap statuses.
   - Report known gaps by status.

2. Add tests.
   - Passing synthetic bundle.
   - Missing required artifact fails.
   - Checksum mismatch fails.
   - Claim-boundary regression fails.
   - Unknown or unreviewed known-gap status fails.

3. Add docs.
   - Document command, expected inputs, report outputs, blocking statuses, and reviewable known gaps.
   - Link the gate from the prerelease checklist.

4. Verify.
   - Run focused readiness tests.
   - Run prerelease bundle tests.
   - Run claim-boundary docs guardrails.
   - Run Ruff on touched Python files.

## Acceptance Checks

- GATE-01: Gate fails when required prerelease artifacts are missing.
- GATE-02: Gate verifies forbidden claim boundaries remain false and representative non-claim wording remains visible.
- GATE-03: Gate reports every known gap as blocking, deferred, unavailable, or diagnostic-only before publishing.
