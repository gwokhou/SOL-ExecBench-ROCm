# Phase 143 Plan

## Goal

Rerun RDNA4 clock-lock and timing evidence after sudoers coverage is available,
while preserving explicit fallback/blocker classification.

## Tasks

1. Verify RDNA4 clock presets and ROCm clock semantics.
   - Compare local `rocm-smi -s` output with configured RDNA4 DPM levels.
   - Use ROCm systems source to confirm `setsclk`/`setmclk` mask semantics.
   - Fix local preset/verification logic if it is inconsistent with observed
     RDNA4 behavior.

2. Rerun live RDNA4 clock-lock evidence.
   - Execute `setperflevel manual`, `setsclk`, `setmclk`, clock queries, reset,
     and `setperflevel auto`.
   - Record command return codes and low-power clock query output.
   - Treat failed set/reset commands as blockers.

3. Rerun timing sidecar generation.
   - Reuse the v1.30 RDNA4 traces and generate v1.31 timing sidecars under a
     new evidence root.
   - Record whether each sidecar is profiler-backed `rocprofv3` kernel
     activity timing or fallback event timing.

4. Close Phase 143.
   - Record evidence summary, verification commands, and claim boundaries.
   - Update requirements and route the milestone to Phase 144.

## Verification

- Focused clock-lock tests pass.
- Ruff passes for changed clock-lock files.
- Live clock-lock evidence is recorded as verified or blocked.
- Timing sidecar summary records the profiler-backed/fallback distribution.
