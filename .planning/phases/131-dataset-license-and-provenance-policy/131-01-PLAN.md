# Phase 131 Plan: Dataset License and Provenance Policy

**Created:** 2026-06-04
**Status:** Ready for execution
**Requirements:** DATA-LIC-01, DATA-LIC-02, DATA-LIC-03, DATA-LIC-04

## Goal

Define enforceable source, license, provenance, and redistribution boundaries
before any migrated dataset artifacts are generated or integrated.

## Tasks

1. Extend machine-readable provenance policy.
   - Add dataset source entries for NVIDIA SOL-ExecBench, FlashInfer Trace,
     generated local migration artifacts, and project-owned ROCm code.
   - Add redistribution classes covering publishable, local-only,
     generated-only, excluded, and release-bundle-blocked boundaries.

2. Add CPU-safe redistribution guardrails.
   - Implement a script that checks staged paths and release bundle roots
     against the policy.
   - Block NVIDIA SOL-ExecBench original and derivative dataset content from
     repository and release redistribution.
   - Integrate release-bundle scanning into prerelease readiness.

3. Update public documentation.
   - Document NVIDIA Evaluation Dataset License boundaries and local-only
     migration policy.
   - Document FlashInfer Trace Apache-2.0 attribution boundaries.
   - Link maintainers to the guardrail commands.

4. Add tests.
   - Cover manifest source and redistribution classification.
   - Cover staged-path/release-root checker behavior without real payloads.
   - Cover prerelease readiness blocking restricted dataset bundle content.
