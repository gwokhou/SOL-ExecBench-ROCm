# Phase 150 Plan: RDNA4 Profiler Replacement Classification

## Goal

Turn Phase 149's all-or-nothing blocker into an auditable classification ledger
so future RDNA4 batches can make measurable progress across the 121 fallback
timing problems.

## Steps

1. Extend profiler timing coverage models with partial and blocked replacement
   counters.
2. Preserve the strict full coverage boundary: only complete `rocprofv3` kernel
   activity sidecars count as `profiler_backed`.
3. Store replacement status, workload counts, trace status counts, and failure
   reason in batch replacement metadata.
4. Update batch resume selection so full-problem classified attempts can be
   skipped while workload-limited smoke artifacts remain retryable.
5. Add CPU-safe tests for coverage classification and resume semantics.
6. Regenerate a layered coverage report using the existing real Phase 149 smoke
   artifact to prove the first blocker is classified as partial rather than
   fallback.

## Acceptance Criteria

- `partial_profiler_backed` and `profiler_blocked` appear in coverage totals
  and status counts when corresponding sidecars exist.
- Partial sidecars do not increment `profiler_backed_problems`.
- Batch summaries distinguish full success, partial profiler-backed attempts,
  and profiler-blocked attempts.
- Existing workload-limited smoke sidecars do not suppress future full-problem
  attempts.
- Relevant pytest and Ruff checks pass.
