<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Matrix Entry to benchmark launcher | Classification decisions determine whether later phases may run benchmark execution. |
| Docker evidence to user claims | Container user-space evidence can be mistaken for native host validation. |
| Compatibility sidecar to canonical trace | Diagnostic fields must remain outside benchmark truth schemas. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-04 | Elevation of Privilege | `host_validated` status | mitigate | Reject Docker scoped entries that claim `host_validated` or `native_host_validated=true`. |
| T-78-05 | Tampering | Mixed-version override | mitigate | Require explicit override parameter and keep clean validation and authority flags false under override. |
| T-78-06 | Repudiation | Benchmark blocking decision | mitigate | Return explicit `benchmark_allowed`, `probes_allowed`, `smoke_allowed`, status, reason code, and reason text. |
| T-78-07 | Tampering | Canonical trace schemas | mitigate | Add public contract guardrails asserting compatibility sidecar keys do not appear in trace JSONL payloads. |
| T-78-SC | Tampering | Package installs | accept | This plan does not add npm, pip, uv, or cargo package installs. |
</threat_model>
