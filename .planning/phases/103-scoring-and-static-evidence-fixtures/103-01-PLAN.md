# Phase 103 Plan 01: Scoring And Static Evidence Fixtures

## Goal

Add focused golden fixture coverage for scoring derivation and explicit static
artifact manifest handling while preserving diagnostic-only sidecar authority.

## Scope

- Add small, deterministic scoring tests for representative operator families
  and AMD SOL v2 status transitions.
- Add optional manifest consumption to static kernel artifact collection.
- Verify sidecar authority fields and existing public contracts remain stable.

## Tasks

1. Add focused scoring fixture tests.
   - Lock formula inputs and confidence for at least one additional
     non-GEMM family through `estimate_bound_work()`.
   - Lock AMD SOL v2 aggregate status and family confidence counts for scored,
     degraded, and unscored representative cases.
   - Prefer existing helper factories and avoid introducing broad fixture churn.

2. Add static evidence manifest support.
   - Add an optional `artifact_manifest_path` argument to
     `collect_static_kernel_artifacts()`.
   - Accept a JSON object with an `artifacts` array containing build-root
     relative paths or objects with a `path` field.
   - Use only root-contained, supported static artifact entries from the
     manifest when it is present.
   - Record an `artifact_manifest` source reference on the sidecar.
   - Preserve current recursive discovery behavior when no manifest is passed.

3. Verify public and diagnostic-only contracts.
   - Assert manifest-driven collection keeps `score_authority` and other
     authority fields false.
   - Run focused scoring/static evidence tests and Ruff.

## Acceptance

- SCORING-01: family-specific golden tests cover representative scoring
  derivation and fallback behavior.
- SCORING-02: AMD SOL v2 status/confidence transitions are directly covered.
- SCORING-03: static kernel evidence can consume an explicit artifact manifest.
- SCORING-04: diagnostic-only authority and public sidecar contracts remain
  unchanged.
