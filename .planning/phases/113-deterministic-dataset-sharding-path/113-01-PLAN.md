# Phase 113 Plan: Define Deterministic Dataset Shard And Merge Semantics

## Goal

Provide an importable, tested sharding contract that future dataset-scale
execution can use without weakening trace provenance.

## Tasks

1. Add a dataset sharding module with shard-plan and merge-result data
   structures.
2. Assign workloads deterministically by input ordinal and stable shard id.
3. Define per-shard trace paths and merge behavior.
4. Test deterministic assignment, ordered merges, duplicate detection, and
   incomplete-shard reporting.

## Acceptance Criteria

- Same workload refs and shard count produce the same shard ids and trace paths.
- Merge output preserves original workload ordering.
- Duplicate traces and missing shard workloads are explicit diagnostics.
- Existing dataset CLI behavior remains unchanged.
