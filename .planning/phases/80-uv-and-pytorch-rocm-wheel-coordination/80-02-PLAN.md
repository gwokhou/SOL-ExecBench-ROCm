<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dependency helper JSON to shell gate | Python dependency status controls whether shell proceeds to Docker build/run. |
| user CLI/env override to debug behavior | User can request mixed-version debug path, but not clean validation authority. |
| unknown Target override to dependency override | Two separate override domains must not grant each other's permissions. |
| preflight decision to Docker execution | Shell must stop before Docker build/run when `benchmark_allowed=false`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-07 | Tampering | helper JSON parsing in `scripts/run_docker.sh` | mitigate | Read bounded fields from dependency helper JSON and gate on `status` plus `benchmark_allowed`; tests assert mixed and unavailable statuses stop before build/run. |
| T-80-08 | Spoofing | false clean validation under mixed dependencies | mitigate | Mixed dependency states print JSON and exit before benchmark execution by default; debug path keeps validation and authority flags false. |
| T-80-09 | Elevation of privilege | `--allow-mixed-version-dependencies` | mitigate | Override name is dependency-specific, passes `--allow-mixed-version-debug`, and tests assert benchmark/authority fields remain false. |
| T-80-10 | Elevation of privilege | `--allow-unknown-target` interaction | mitigate | Keep unknown Target override separate from dependency mismatch override; tests assert unknown-target override does not permit mixed dependencies. |
| T-80-11 | Denial of service | shell continuing after dependency preflight says blocked | mitigate | Script exits nonzero before Docker build/run when dependency JSON has blocked status or `benchmark_allowed=false`; tests assert no Docker command text appears in blocked output. |
| T-80-SC | Tampering | package installs | accept | No package installs; script integration consumes existing Phase 80 helper and does not run uv, pip, or lockfile updates. |
</threat_model>
