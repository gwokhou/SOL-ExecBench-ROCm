# Phase 145 Plan

## Goal

Classify or reproduce the 12 RDNA4 `missing_trace` workload records from the
Phase 138 ready-subset execution.

## Tasks

1. Build missing-trace inventory.
   - Extract all `closure_status == "missing_trace"` records from
     `out/rdna4-full-dataset/run/execution_closure.json`.
   - Preserve problem ID, row index, workload UUID, trace ref, and timing
     evidence ref.

2. Cross-check trace artifacts.
   - Verify each target problem has a `traces.json`.
   - Check whether the target workload UUID exists in the merged traces.
   - Count existing traces per affected problem.

3. Attach execution evidence.
   - Locate the per-workload shard CLI log for each missing row.
   - Classify the root cause from bounded log evidence.

4. Close the requirement.
   - Write machine-readable and human-readable triage outputs.
   - Update v1.31 planning status and route to Phase 146.

- Exactly 12 missing-trace closure records are present.
- Every missing-trace row has a non-unknown triage classification.
- Every classified row has a concrete evidence ref.
- The conclusion preserves the failed-workload denominator and does not upgrade
