# Plan 95-01: Runtime Helper Extraction

## Goal

Move staged problem loading, reference loading, entry-point parsing, user
function loading, native ROCm language detection, and dynamic extension blocking
from the generated eval driver into an importable runtime helper module.

## Tasks

| Task | Files | Action | Verification |
| --- | --- | --- | --- |
| 1 | `src/sol_execbench/core/bench/eval_runtime.py` | Add helpers for staged problem loading, entry-point parsing, reference-module loading, native ROCm solution detection, dynamic cpp-extension load blocking, and user function resolution. | Unit tests cover JSONL loading, missing definition error, entry-point parsing, reference loading, Python user function loading, native missing `.so`, and blocked extension load. |
| 2 | `src/sol_execbench/driver/templates/eval_driver.py` | Replace inline loading/import/blocking code with helper calls while preserving template responsibilities. | Driver smoke tests pass. |
| 3 | `tests/sol_execbench/core/bench/test_eval_runtime.py` | Add focused CPU-safe helper tests. | Targeted test file passes. |

## Constraints

- Preserve stdout redirection and staging path insertion order in the template.
- Preserve existing exception messages where driver tests or users rely on them.
- Do not move the workload correctness/timing loop in this plan.
