# Phase 107 Plan 01: File-Based Staged User Imports

## Goal

Load Python solution entry files through unique file-based module specs so
ordinary module-name collisions cannot affect evaluation.

## Tasks

1. Add helpers in `src/sol_execbench/core/bench/eval_runtime.py` for unique
   staged module names and synthetic package registration.
2. Replace the Python branch of `load_user_function()` with file-based
   `importlib.util.spec_from_file_location()` loading.
3. Preserve native ROCm loading and staging `sys.path` compatibility.
4. Add tests for simple-file collisions and package-like path collisions in
   `tests/sol_execbench/core/bench/test_eval_runtime.py`.
5. Run focused eval-runtime tests and eval-driver subprocess tests.

## Verification

- `uv run pytest tests/sol_execbench/core/bench/test_eval_runtime.py -q`
- `uv run pytest tests/sol_execbench/driver/test_eval_driver.py -q`
