# Phase 147 Plan

avoidable evaluator memory pressure and the all-fallback timing sidecars. Only
record a residual boundary after the attempt is backed by evidence.

## Tasks

1. Reduce avoidable eval-driver memory retention.
   - Avoid unconditional reference-output clones when the output does not
     alias input storage.
   - Release correctness outputs before timing starts.
   - Preserve correctness behavior for aliasing reference outputs.

2. Validate the memory change.
   - Add a CPU-safe regression test for aliasing reference outputs.
   - Rerun at least one known RDNA4 memory-failed workload in an isolated
     memory-capped unit.

3. Investigate timing sidecar fallback.
   - Parse all v1.31 timing sidecars.
   - Confirm whether fallback is caused by `rocprofv3` availability, parser
     failure, or source-language timing policy.
   - Record the root cause and what evidence would be required for
     profiler-backed timing.

4. Update milestone closure.
   - Add `RDNA4-FU-TIME-04`.
   - Update `RDNA4-FU-CLOSE-01` to require attempted closure work.
   - Update roadmap, state, audit counts, and the closure handoff.
