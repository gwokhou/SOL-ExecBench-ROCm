<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> filesystem | User-supplied paths are opened only when explicitly provided to the report script. |
| local JSON files -> core report builder | Sidecar JSON payloads are untrusted local input and are normalized by Plan 87-01 strict models. |
| script output -> researcher interpretation | JSON/Markdown artifacts may be mistaken for validation or authority without guardrails. |
| public contracts -> downstream users | Canonical schemas and primary CLI help are stable public surfaces that must not absorb diagnostic sidecar fields. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87-02-01 | Tampering | `scripts/report_amd_bound_sanity.py` input loading | mitigate | Load only explicit JSON paths, delegate normalization/checksum/status logic to the strict core builder, and avoid implicit filesystem scans. |
| T-87-02-02 | Repudiation | script determinism | mitigate | Require fixed-output JSON/Markdown helpers and test repeated runs with fixed `created_at` for byte-identical output. |
| T-87-02-03 | Information Disclosure | script output | mitigate | Emit bounded refs/checksums and summaries only; do not embed raw traces, full Matrix payloads, credentials, environment scans, or hardware probe output. |
| T-87-02-04 | Elevation of Privilege | public CLI and schema surfaces | mitigate | Guardrail tests prove the diagnostic report is not exposed as a primary benchmark CLI option and does not enter canonical schemas or score eligibility contracts. |
| T-87-02-05 | Spoofing | validation/authority claims | mitigate | Guardrail tests assert claim-boundary fields stay false and Markdown keeps validation/equivalence/authority claims explicitly negative. |
| T-87-02-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager install tasks are planned; use existing Python stdlib and Plan 87-01 helpers only. |
