# Phase 193-03 Plan: Wire Measured Baseline Provenance into Official Score Gate (BASE-03)

**Requirement:** BASE-03
**Wave:** 2 (**Depends on:** 193-02 — consumes `baseline_coverage.BaselineCoverageReport`)
**Files modified:** `src/sol_execbench/core/scoring/official_score.py`,
`tests/sol_execbench/core/evidence/test_official_score_evidence.py`

## Objective

Make measured baseline provenance an authoritative baseline source for the
official score gate, gated by the BASE-02 coverage-validation report, while
keeping `ScoringBaselineArtifact` (release-scoped) and the measured baseline
registry as SEPARATE schemas. The gate stays STAGING — it is NOT wired into any
CLI/runner/sidecar run path in this phase (emission belongs to Phase 194).

## Decisions Covered

- D-10: Add `measured_baseline_registry` as an accepted value in
  `DEFAULT_OFFICIAL_BASELINE_SOURCES` AND accept a coverage-validation report as
  a gate precondition parameter on `official_score_from_amd_native_score` /
  `build_official_score_suite_evidence`. Both changes are required.
- D-11: When coverage validation fails, the gate emits an umbrella blocker
  `baseline_coverage_failed` AND propagates the specific coverage reason codes
  (e.g. `baseline_hardware_mismatch`) so HIP sees precise failure reasons.
- D-12: KEEP `official_score.py` STAGING in 193 — do NOT wire into a CLI
  command, runner, or sidecar writer. The module docstring's STAGING notice is
  preserved/updated; the aggregation-policy precondition remains an explicit
  parameter.
- D-13: Keep `ScoringBaselineArtifact` and the measured baseline registry as
  SEPARATE schemas; both remain accepted `baseline_source` values. Do not merge.
- D-08 (consistency): a test asserts the coverage module's `missing_baseline` /
  `placeholder_baseline` literals equal official_score's
  `MISSING_BASELINE_BLOCKER` / `PLACEHOLDER_BASELINE_BLOCKER` constants.

## Implementation Steps

1. Read before editing:
   - `src/sol_execbench/core/scoring/official_score.py` (current STAGING gate,
     `DEFAULT_OFFICIAL_BASELINE_SOURCES`, blocker constants,
     `_official_score_blockers`, `official_score_from_amd_native_score`,
     `build_official_score_suite_evidence`, and the module docstring's two
     unresolved-decision note).
   - `src/sol_execbench/core/evidence/baseline_coverage.py` (produced by 193-02:
     `BaselineCoverageReport`, state/reason-code constants).
   - `src/sol_execbench/core/scoring/baseline_artifact.py` (ScoringBaselineArtifact
     — kept separate, not merged).
   - `tests/sol_execbench/core/evidence/test_official_score_evidence.py`
     (existing gate tests to extend).
2. In `official_score.py`:
   - Add `BASELINE_COVERAGE_FAILED_BLOCKER = "baseline_coverage_failed"`.
   - Extend `DEFAULT_OFFICIAL_BASELINE_SOURCES` to include
     `"measured_baseline_registry"` alongside `"scoring_baseline"` (D-13: both
     accepted).
   - Add an optional keyword parameter
     `coverage_report: BaselineCoverageReport | None = None` (import the type
     from `sol_execbench.core.evidence.baseline_coverage`) to both
     `official_score_from_amd_native_score` and
     `build_official_score_suite_evidence`. Thread it through to
     `_official_score_blockers`.
   - In `_official_score_blockers`, when `coverage_report` is supplied AND its
     report is not fully confirmed (i.e. any entry is in a non-confirmed state),
     append `BASELINE_COVERAGE_FAILED_BLOCKER` plus propagate the report's
     non-confirmed reason codes (deduplicated, preserving stable order). When
     `coverage_report` is `None`, gate behavior is unchanged (backward
     compatible).
   - Update the module docstring's "unresolved decisions" note: the
     baseline-source coverage (point 2) is now resolved for
     `scoring_baseline` and `measured_baseline_registry`; aggregation policy
     (point 1) remains an explicit parameter. Keep the STAGING notice (D-12).
3. Update `tests/sol_execbench/core/evidence/test_official_score_evidence.py`:
   - **Existing coverage gate test** (referenced by the module docstring as
     `test_official_score_baseline_source_sets_cover_amd_score_universe`) MUST
     still pass after extending the accepted source set — update its expectation
     if it enumerates the accepted sources explicitly.
   - Add: a workload backed by `measured_baseline_registry` with a fully
     `confirmed` coverage report produces a non-null official score (no
     `baseline_coverage_failed` blocker).
   - Add: a coverage report containing a `mismatched`/`stale`/`missing`/
     `placeholder` entry adds `baseline_coverage_failed` AND the specific
     propagated reason code to `blocker_reason_codes`, and forces the official
     score to `None`.
   - Add: when `coverage_report` is `None`, behavior is unchanged (no
     `baseline_coverage_failed` blocker) — backward compatibility.
   - Add: a consistency test asserting
     `baseline_coverage.MISSING_BASELINE_CODE == official_score.MISSING_BASELINE_BLOCKER`
     and `baseline_coverage.PLACEHOLDER_BASELINE_CODE == official_score.PLACEHOLDER_BASELINE_BLOCKER`
     (D-08).
   - Add: `ScoringBaselineArtifact` and `measured_baseline_registry` both remain
     accepted `baseline_source` values; a test confirms neither is rejected
     when coverage is confirmed (D-13).
