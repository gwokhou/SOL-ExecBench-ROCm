# 96-01 Plan: Bound Graph Classification Split

## Objective

Move call-name classification and related operation-family taxonomy out of
`amd_bound_graph.py` into an importable helper module with direct tests.

## Tasks

1. Add `sol_execbench.core.scoring.amd_bound_classification` for
   `CallClassification`, call classifier tables, movement-kind classification,
   and dtype method targets.
2. Update `amd_bound_graph.py` to delegate classification decisions to the new
   helper while preserving existing graph node families, attributes, warnings,
   and serialized payloads.
3. Add focused tests that verify representative call names map to the expected
   families and movement/dtype helper behavior.
4. Run AMD bound graph tests and Ruff on touched files.

## Acceptance

- Public imports from `amd_bound_graph.py` remain compatible.
- Existing AMD bound graph tests pass.
- New tests can validate classification behavior without building a full graph.
