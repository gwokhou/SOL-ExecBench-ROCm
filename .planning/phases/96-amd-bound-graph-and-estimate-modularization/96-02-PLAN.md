# 96-02 Plan: Estimate Family Dispatch Split

## Objective

Make estimate dispatch explicitly grouped by operation family so formula
ownership is easier to inspect and test without scanning the full estimate
module.

## Tasks

1. Add `sol_execbench.core.scoring.amd_bound_estimate_families` for dispatch
   family grouping and public helper functions.
2. Update `amd_bound_estimates.py` to use the helper for `_estimate_node`
   dispatch while preserving every existing formula output.
3. Add focused tests for representative attention, GEMM/linear, pointwise,
   movement, and unsupported family dispatch groups.
4. Run AMD bound estimate tests and Ruff on touched files.

## Acceptance

- Existing estimate outputs remain schema-compatible and test-compatible.
- New helper tests can identify the formula group for a family without running
  full graph estimation.
