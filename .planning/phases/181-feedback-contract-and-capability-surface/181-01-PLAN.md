# Phase 181 Plan 01: Optional Feedback Contract Surface

**Phase:** 181 - Feedback Contract and Capability Surface
**Status:** Ready
**Requirements:** CNTR-01, CNTR-02, CNTR-03

## Goal

Expose optional SOL agent-feedback/profile-summary capability discovery and
document authority boundaries without changing canonical Trace JSONL semantics.

## Tasks

1. Update `src/sol_execbench/core/data/contract.py`
   - Add `agent_feedback.sidecar.v1`.
   - Add `profile_summary.sidecar.v1`.
   - Add source-boundary claims covering diagnostic-only authority and HIP/SOL
     ownership boundaries.
   - Preserve `SOL_EXECBENCH_CONTRACT_VERSION == "1.0"`.

2. Update `tests/sol_execbench/test_contract.py`
   - Add optional capability assertions.
   - Assert feedback/profile-summary capabilities do not appear in trace field
     requirements, correctness fields, timing fields, or scoring fields.
   - Assert boundary claim wording covers diagnostic-only and non-authority
     semantics.

3. Add evaluator contract documentation
   - Create `docs/EVALUATOR-CONTRACT.md`.
   - Explain capability discovery, canonical trace boundaries, optional
     feedback sidecars, and downstream HIP responsibilities.
   - Link the document from `docs/DEVELOPMENT.md`.

4. Verify
   - Run `uv run pytest tests/sol_execbench/test_contract.py`.
   - Run `uv run sol-execbench contract --json`.

## Success Criteria

- `sol-execbench contract --json` advertises optional agent-feedback and
  profile-summary capability tokens without adding required trace fields.
- Documentation says feedback/profile-summary sidecars are diagnostic-only and
  not correctness, timing, score, evidence-tier, release-gate, cutover,
  paper-parity, or leaderboard authority.
- Contract tests prove canonical Trace JSONL field groups and status semantics
  remain unchanged.
