# Plan 94-01: Selection And Run-State Helpers

## Goal

Move dataset discovery, ready-subset workload selection, trace indexing, trace
status mapping, and requested-evidence normalization out of
`scripts/run_dataset.py` into importable helpers under
`sol_execbench.core.dataset`.

## Tasks

| Task | Files | Action | Verification |
| --- | --- | --- | --- |
| 1 | `src/sol_execbench/core/dataset/run_state.py` | Add typed helper functions for problem discovery, workload keying, workload-row reads, selected workload filtering/capping/missing rows, trace indexing/status, and requested evidence requirements. | New helper tests cover category discovery, ready-subset selection, cap filtering, missing refs, duplicate refs, trace maps, and evidence flags. |
| 2 | `scripts/run_dataset.py` | Replace inline helper implementations with imports or thin compatibility wrappers while preserving internal call behavior. | Existing run_dataset execution-closure tests pass. |
| 3 | `tests/sol_execbench/test_dataset_run_state.py` | Add focused unit tests that exercise helper contracts without importing the script. | Targeted test file passes. |

## Constraints

- Preserve CLI behavior and output filenames.
- Preserve workload selection order and duplicate suppression.
- Preserve private helper names in `scripts/run_dataset.py` when existing tests
  import them indirectly.
