# Phase 146 Plan

## Goal

Classify the 146 failed RDNA4 workload records and confirm that public claims
remain bounded after follow-up triage.

## Tasks

1. Build failure inventory.
   - Extract `attempted_failed` and `missing_trace` closure records.
   - Preserve trace status, problem ID, row index, workload UUID, trace ref,
     and any CLI-log ref.

2. Classify root causes.
   - Inspect trace `evaluation.log` text and Phase 145 missing-trace evidence.
   - Group failures into stable classes with representative examples.
   - Record likely root cause and next action per class.

3. Reassess claim boundaries.
   - Keep all 146 failures denominator-visible.
   - Update public docs to mention v1.31 classifications without changing pass
     counts or upgrading validation authority.

4. Close the milestone phase.
   - Write Phase 146 summary and verification artifacts.
   - Mark `RDNA4-FU-FAIL-01` and `RDNA4-FU-CLAIM-01` complete.
   - Route milestone to audit/completion.
