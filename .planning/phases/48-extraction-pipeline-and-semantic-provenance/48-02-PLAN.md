<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Definition/Workload -> graph builder | Canonical problem data enters FX/AST extraction. |
| BoundGraph/estimates -> derivation sidecar | Internal extraction artifacts are transformed into semantic provenance. |
| candidate solution -> derivation pipeline | Submitted solution code must remain outside the derivation boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-04 | Elevation of Privilege | `build_solar_derivation_evidence` | mitigate | Accept only `Definition` and `Workload`; never accept or execute candidate solution objects, paths, or submitted source code. |
| T-48-05 | Tampering | tensor provenance builder | mitigate | Copy graph and estimate evidence into immutable dataclasses and parse serialized output through strict validators. |
| T-48-06 | Spoofing | source attribution | mitigate | Emit explicit `SolarEvidenceSource.kind` and `detail` values so evidence cannot appear formula-backed without a visible source. |
| T-48-SC | Tampering | package installs | accept | No package-manager install tasks are present; Phase 48 uses existing extraction stack. |
</threat_model>
