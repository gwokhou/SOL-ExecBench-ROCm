<objective>
Implement deterministic benchmark-defined custom input generation readiness for Phase 170.

Purpose: The 55 L1/L2 custom-input readiness blockers cannot be attempted safely on ROCm while input generation is either blocked statically or only surfaced as a generic runtime error. This plan makes evaluator input assembly deterministic and schema-validated, then moves readiness classification from a blanket custom-input blocker to a precise executable-or-blocked decision.

Output: One implementation path for custom input execution support, validation/provenance evidence, readiness reclassification, and CPU-safe regression coverage. This phase does not recompute RDNA4 coverage; Phase 171 consumes these capabilities.
</objective>

<threat_model>
Inputs generated from dataset reference code execute inside the existing staged evaluator trust boundary, not inside the main CLI process. This plan must not widen that boundary: reference/custom-input code stays staged with the problem, canonical traces remain strict JSON, and generated input evidence must not serialize tensor payloads or proprietary dataset content.

Risks to control:
- High: malformed custom input functions could mask candidate failures as reference/runtime failures. Mitigation: validate generated inputs before reference/candidate calls and emit input-generation-specific diagnostics.
- Medium: custom generation can perturb global RNG state and make correctness/timing evidence irreproducible. Mitigation: derive a stable per-workload seed and restore PyTorch RNG state after generation.
- Medium: readiness reclassification could over-promote malformed custom datasets. Mitigation: only remove the blanket `custom_input_blocked` path when `custom_inputs_entrypoint` exists and all workload custom inputs are structurally supported; malformed entries remain explicit blockers.
- Low: provenance sidecars/logs could leak generated tensor contents. Mitigation: record seed, entrypoint, expected/actual metadata, and failure class only, not tensor values.
</threat_model>
