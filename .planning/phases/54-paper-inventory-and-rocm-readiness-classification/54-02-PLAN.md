<objective>
Implement static ROCm readiness classification from inventory artifacts without
running GPU evaluation or claiming execution success.
</objective>

<threat_model>
- Misleading readiness: `ready` means ready to attempt local ROCm execution,
  not passed, scored, or validated.
- Unsafe static scan: avoid broad NVIDIA string blocking that treats documented
  ROCm `torch.cuda` compatibility APIs as NVIDIA-only blockers.
- Asset substitution: missing custom/safetensors assets must be blockers, not
  random inputs.
</threat_model>
