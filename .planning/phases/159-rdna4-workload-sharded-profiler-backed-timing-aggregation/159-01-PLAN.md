# Phase 159 Plan

## Goal

Implement workload-sharded profiler-backed timing aggregation so RDNA4
profiler OOM and timeout blockers are solved by bounded execution units and
deterministic aggregation instead of longer full-problem runs.

## Tasks

1. Define a workload profiler manifest schema.
   - Include problem id, expected workload count, workload ids or offsets,
     runner version, source dataset root, trace paths, kernel row counts,
     profiler backend, kernel timing totals, status, failure reason, and
     retryability.

2. Add workload-sharded execution mode to the profiler batch runner.
   - Execute one workload per profiler session by default.
   - Reuse existing `--workload-offset`, temp-root, skip-list, and blocked
     sidecar controls.
   - Write or update the manifest after each workload so interrupted runs can
     resume without repeating completed slices.

3. Implement problem-level aggregation.
   - Promote to `profiler_backed` only when every expected workload is present,
     passed, `profiler_collected=true`, `backend=rocprofv3`, and has kernel
     timing rows.
   - Preserve source trace paths and per-workload timing details in the
     aggregate sidecar.
   - Keep incomplete manifests as `partial_profiler_backed`, not full coverage.

4. Update coverage reporting.
   - Teach the coverage report to consume aggregated workload-sharded sidecars.
   - Keep diagnostic slices separate from complete aggregated replacements.
   - Report partial manifest counts and blocked workload reasons explicitly.

5. Add CPU-safe tests.
   - Manifest resume does not rerun completed workloads.
   - Aggregation rejects missing, failed, fallback, or trace-less workloads.
   - Aggregation accepts a complete all-workload profiler-backed manifest.
   - Coverage counts accepted aggregate sidecars as `profiler_backed`.

6. Run one real RDNA4 acceptance path.
   - Use existing `L1/037_flux_feedforward_gelu_approximate` workload evidence
     or rerun missing slices through the new manifest path.
   - Produce one aggregate problem-level profiler-backed timing sidecar.
   - Regenerate the 235-problem coverage summary and verify the problem moves
     from partial/blocked evidence to full `profiler_backed`.

## Non-Goals

- Do not upgrade public RDNA4 timing authority claims in this phase.
- Do not count partial or diagnostic-only workload slices as full problem
  profiler-backed coverage.
- Do not rely on a single full-problem high-timeout run as the primary closure
  mechanism.
