# Plan 167-01: RDNA4 Clock-Lock Evidence

## Objective

Record host-level RDNA4 clock-lock, reset, driver, and policy evidence so
benchmark-grade timing claims can distinguish profiler evidence from
authoritative timing evidence.

## Scope

- Capture `rocm-smi` pre-lock hardware, driver, clock, power, temperature, and
  utilization state.
- Exercise operator clock-control commands for manual performance mode, SCLK,
  MCLK, and reset.
- Record post-command observed clock state and reset state.
- Document the claim boundary when requested clock masks succeed but observed
  clocks do not prove stable maximum-frequency operation.

## Acceptance Criteria

- Pre/run/post evidence includes GPU architecture, driver, performance level,
  supported clocks, observed clocks, power, temperature, and utilization.
- Reset evidence proves the host was returned to automatic clock policy after
  the experiment.
- Documentation states that `rocprofv3` timing is necessary but insufficient
  for authoritative timing without stable clock-lock evidence.
- Phase 168 can cite a single clock-lock artifact without upgrading RDNA4
  timing claims beyond the evidence.

## Verification

- Run read-only `rocm-smi` commands before and after clock-control attempts.
- Run `sudo rocm-smi --setperflevel manual`, `--setsclk`, `--setmclk`, and
  `--resetclocks` through approved operator escalation.
- Confirm final `rocm-smi` state is `Performance Level: auto`.
