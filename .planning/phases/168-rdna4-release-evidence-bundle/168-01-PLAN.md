# Plan 168-01: RDNA4 Release Evidence Bundle

## Objective

Package the current RDNA4 evidence chain into a release-grade manifest and
claim-upgrade review without rerunning heavyweight benchmark workloads.

## Scope

- Reference full dataset execution, timing, profiler coverage, blocker,
  derived-score, AMD SOL/SOLAR, and clock evidence artifacts.
- Record checksums for the top-level files needed to audit the bundle.
- State which RDNA4 claims are supported and which remain blocked.
- Preserve deferred boundaries for CDNA3/MI300X, CDNA4, paper parity, upstream
  SOLAR equivalence, authoritative timing, score authority, and leaderboard
  authority.

## Acceptance Criteria

- The release manifest identifies evidence paths, counts, checksums, hardware
  metadata, and claim authority fields.
- The claim review makes no upgrade that is not supported by recorded evidence.
- Phase 168 can be audited from repository artifacts plus the referenced local
  `out/` evidence directories.
