# Phase 194-01 Plan: Evaluator Contract Confirmed-Evidence Surface (GATE-01)

**Requirement:** GATE-01
**Wave:** 1 (independent)
**Files modified:** `src/sol_execbench/core/data/contract.py`, `tests/sol_execbench/test_contract.py`

## Objective

Extend `build_evaluator_contract()` so the evaluator contract advertises the
confirmed benchmark evidence capabilities (official score, measured baseline
coverage), records confirmed ownership boundaries (keeping diagnostic sidecars
diagnostic), and exposes the stable blocker reason-code vocabulary HIP must
consume for cutover-gate decisions.

## Decisions Covered

- D-01: Add confirmed capabilities `official_score.evidence: "confirmed"` and
  `measured_baseline.coverage: "confirmed"` (keep `profiling.evidence: "optional"`
  and diagnostic sidecar capabilities unchanged).
- D-02: Add confirmed boundaries `official_score` and `measured_baseline` with
  `authority: "confirmed"`; keep all diagnostic sidecar boundaries at
  `authority: "diagnostic"`.
- D-03: Advertise the stable blocker reason-code vocabulary on the contract.
- D-04: Keep schema version `sol_execbench.evaluator_contract.v2` (additive).

## Implementation Steps

1. Read `src/sol_execbench/core/data/contract.py` and
   `tests/sol_execbench/test_contract.py` before editing (note the stability and
   stale-token guard tests: `test_evaluator_contract_versions_are_stable`,
   `test_evaluator_contract_capability_keys_do_not_use_schema_ids`,
   `test_evaluator_contract_declares_required_capabilities`,
   `test_active_source_and_fixture_text_avoid_stale_sidecar_capability_tokens`).
2. In `contract.py`:
   - Add to `capabilities`: `"official_score.evidence": "confirmed"` and
     `"measured_baseline.coverage": "confirmed"`.
   - Add to `boundaries`:
     `{"owner": "sol", "scope": "official_score", "authority": "confirmed"}` and
     `{"owner": "sol", "scope": "measured_baseline", "authority": "confirmed"}`.
   - Add a new additive field `confirmed_evidence_blockers: list[str]` to
     `EvaluatorContract` (with a docstring) populated with the stable blocker
     codes: `missing_score`, `missing_measured_latency`, `missing_baseline`,
     `placeholder_baseline`, `missing_sol_bound`, `missing_aggregation_policy`,
     `baseline_coverage_failed`. Source the literals from
     `sol_execbench.core.scoring.official_score` constants where available (import
     is fine: `data` already depends on scoring-adjacent modules) — or mirror
     them as literals with a consistency test.
   - Do NOT bump `SOL_EXECBENCH_CONTRACT_SCHEMA_VERSION` (stays v2).
3. In `tests/sol_execbench/test_contract.py`:
   - Assert `official_score.evidence` and `measured_baseline.coverage` are in
     `capabilities` with value `"confirmed"`.
   - Assert the `boundaries` list contains `official_score` and
     `measured_baseline` with `authority: "confirmed"`, and that diagnostic
     sidecar boundaries remain `authority: "diagnostic"`.
   - Assert `confirmed_evidence_blockers` contains the expected codes.
   - Ensure `test_evaluator_contract_versions_are_stable` still passes (versions
     unchanged). Ensure
     `test_evaluator_contract_capability_keys_do_not_use_schema_ids` still passes
     (new keys are not schema IDs).
4. Update `docs/EVALUATOR-CONTRACT.md` is handled in Plan 194-04, but the new
   capability tokens MUST also appear in a doc/fixture so the stale-token guard
   (`test_active_source_and_fixture_text_avoid_stale_sidecar_capability_tokens`)
   does not flag them as stale. Coordinate with 194-04; at minimum reference the
   new capabilities in `docs/EVALUATOR-CONTRACT.md` within this plan's commit if
   the guard would otherwise fail.
