# Phase 149 Plan: RDNA4 Profiler-Backed Timing Batch Replacement

## Objective

Add an explicit RDNA4 batch replacement runner for the 121 fallback timing
sidecars. The runner should read the Phase 148 coverage ledger, select
`timing_fallback` problems, profile staged `eval_driver.py` directly under
`rocprofv3`, and write profiler-backed timing sidecars in a coverage-compatible
layout.

## Tasks

1. Add a forced `rocprofv3` kernel activity timing policy for replacement
   batches without changing the default source timing policy.
2. Add a batch script that:
   - builds inventory/readiness/coverage,
   - selects `timing_fallback` problems,
   - supports `--limit`, `--resume`, and `--only-problem`,
   - stages each problem with `ProblemPackager`,
   - runs `rocprofv3 --kernel-trace` against `python eval_driver.py`, and
   - writes replacement sidecars only when profiler collection succeeds.
3. Add CPU-safe tests for target selection, replacement sidecar writing, and
   failure handling.
4. Verify with a small fake-runner test suite and, if practical, run a small
   real RDNA4 limit after implementation.
