# Phase 108 Plan 01: Native Compile Option Validation

## Goal

Reject dangerous native compile options before they reach
`torch.utils.cpp_extension.load()`, while preserving documented ROCm/HIP
extension flags.

## Tasks

1. Add `CompileOptions` validation for all flag lists in
   `src/sol_execbench/core/data/solution.py`.
2. Reject response files, path-injection flags, rpath/dynamic-loader controls,
   plugin/sysroot controls, and split path options.
3. Add schema tests in `tests/sol_execbench/core/data/test_solution.py`.
4. Add build-template regression tests in
   `tests/sol_execbench/driver/test_build_ext.py`.
5. Run focused schema/build-template tests plus Ruff.

## Verification

- `uv run pytest tests/sol_execbench/core/data/test_solution.py -q`
- `uv run pytest tests/sol_execbench/driver/test_build_ext.py -q`
- `uv run ruff check src/sol_execbench/core/data/solution.py tests/sol_execbench/core/data/test_solution.py tests/sol_execbench/driver/test_build_ext.py`
