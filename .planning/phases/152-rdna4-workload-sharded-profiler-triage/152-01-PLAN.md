# Phase 152 Plan

## Goal

Add workload-sharded RDNA4 profiler timing controls so problematic fallback
targets can be triaged without repeatedly running every workload in the
problem.

## Tasks

1. Add `--workload-offset` to the profiler timing batch runner.
2. Record workload slice metadata in replacement sidecars.
3. Preserve resume semantics so workload slices remain retryable and cannot be
   mistaken for full-problem classifications.
4. Add CPU-safe tests for workload offset and slice metadata.
5. Run one bounded real workload slice to prove the path works without OOM.

## Claim Boundary

Workload slices are diagnostic profiler evidence only. They do not satisfy full
profiler-backed timing coverage.
