# Phase 192-01 Plan: Official Score Evidence Contract

## Objective

Add an independent official score evidence contract so SOL can expose benchmark
score authority separately from diagnostic trace speedups and existing
AMD-native derived score evidence.

## Decisions Covered

- D-01: Keep `official_score_evidence` separate from `amd_native_score.v1`.
- D-02: Treat `amd_native_score.v1` as an input, not as the authoritative official
  score contract.
- D-03: Keep diagnostic Trace JSONL `speedup_factor` distinct from official
  benchmark score.
- D-04: Require measured latency, an official measured baseline, SOL/SOLAR bound
  evidence, and an explicit aggregation policy before emitting a non-null
  official score.
- D-05: Emit `null` official scores with stable blocker reason codes whenever
  required inputs are missing.
- D-06: Include suite-level scored/unscored counts and input refs.
- D-07: Block `reference_latency_ms` and placeholder/reference baseline fallback
  for confirmed official score claims.
- D-08: Return a stable placeholder/reference baseline blocker instead of a
  provisional official score.
- D-09: Keep provisional AMD-native scores separate and classify them as
  non-authoritative unless backed by official measured baseline evidence.
- Preserve diagnostic sidecar boundaries; profile and feedback artifacts remain
  non-authoritative for official scoring.

## Implementation Steps

1. Add `src/sol_execbench/core/scoring/official_score.py`.
   - Define `OFFICIAL_SCORE_SCHEMA_VERSION`.
   - Define stable blocker reason codes for missing score inputs,
     missing official baseline, placeholder baseline, missing SOL bound, and
     missing aggregation policy.
   - Add per-workload and suite-level dataclasses with `to_dict()` output.
   - Build official score evidence from existing `AmdNativeScore` objects while
     retaining citations to trace, timing, baseline, SOL bound, hardware model,
     and derived evidence refs.

2. Export the official score API from `sol_execbench.core.scoring`.

3. Add focused unit tests in `tests/sol_execbench/test_official_score_evidence.py`.
   - Valid scoring-baseline-backed input produces a non-null official score.
   - `reference_latency` or placeholder baseline blocks official score.
   - Missing aggregation policy blocks official score.
   - Missing numeric inputs/SOL bound produce stable blockers.
   - Suite reports expose scored/unscored counts and blocker summaries.

4. Update evaluator-facing documentation.
   - Document that official score evidence is the only score-authoritative
     report for confirmed benchmark claims.
   - Document that AMD-native score and diagnostic speedups are supporting
     inputs only unless converted through the official evidence gate.

## must_haves

- D-01: Introduce a separate `official_score_evidence` schema/report instead of
  extending `amd_native_score.v1`.
- D-02: Existing AMD-native scores may be referenced as inputs, but the official
  evidence artifact is the only confirmed score authority.
- D-03: Diagnostic Trace JSONL `speedup_factor` remains distinct from official
  benchmark score fields.
- D-04: A non-null official score requires measured candidate latency, official
  measured baseline latency, SOL/SOLAR bound evidence, and aggregation policy.
- D-05: Missing required inputs produce `null` official scores plus stable
  blocker reason codes.
- D-06: Suite-level evidence reports scored count, unscored count, and input refs.
- D-07: `reference_latency_ms` and placeholder/reference baseline fallback cannot
  satisfy confirmed official score baseline authority.
- D-08: Placeholder/reference baseline use produces a stable blocker reason code.
- D-09: Provisional AMD-native scores remain available separately and become
  authoritative only after the official evidence gate confirms measured baseline
  backing.

## Out Of Scope

- Measured baseline artifact schema expansion is handled in Phase 193.
- HIP cutover-gate bundle integration is handled in Phase 194.
- Real GPU validation remains deferred until the full evidence path is wired.
