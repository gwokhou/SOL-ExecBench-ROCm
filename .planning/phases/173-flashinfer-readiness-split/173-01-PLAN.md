# Plan 173-01: FlashInfer Readiness Split

## Objective

Split FlashInfer-Bench readiness from category-only blocking into semantic
readiness classes so PyTorch-native simple workloads can move out to ready, while
paged/MLA/ragged/runtime-dependent workloads remain explicitly blocked until
dedicated ROCm compatibility work exists.

## Background

The previous checker treated all `FlashInfer-Bench` problems as runtime blocked.
Current semantics show 26 problems with mixed intent. The simple family includes
`fused_add_rmsnorm`, `gemm`, and `rmsnorm` paths that can execute via existing
PyTorch semantics, while runtime families need dedicated blockers such as paged
decode/prefill, ragged prefill, MLA paging, MoE fp8 block-scale, or unknown
runtime dependence.

## must_haves

### truths

- Category-level `flashinfer_runtime_assumption` blocking is replaced by per-problem
  semantic taxonomy.
- Simple PyTorch-compatible problems are ready only when runtime hints are absent
  from migrated reference text.
- Runtime-dependent problems are blocked with specific bucket reasons and actionable
  next actions.

### deliverables

- `FLASHINFER_SIMPLE_REFERENCE_TOKENS` for direct simple-case detection.
- `FLASHINFER_RUNTIME_BUCKETS` and runtime-to-reason mapping for
  paged/ragged/MLA/moe runtime groups.
- `flashinfer_*` readiness reason and blocker evidence generation for each
  runtime bucket.
- CPU-safe regression tests covering bucket split and blocker continuity.

## Task 1: Add deterministic FlashInfer runtime taxonomy in readiness

In `readiness.py`, define simple-token and runtime-bucket constants and classification
helpers:

- Add `FLASHINFER_SIMPLE_REFERENCE_TOKENS` for `fused_add_rmsnorm`, `gemm`,
  and `rmsnorm`.
- Add `FLASHINFER_RUNTIME_BUCKETS` for `paged_decode`, `paged_prefill`,
  `ragged_prefill`, `mla_paged`, `moe_fp8_block_scale`.
- Add `FLASHINFER_RUNTIME_BUCKET_TO_REASON` for bucket-coded reason strings.
- Add `_flashinfer_reference_is_runtime_dependent` and `_flashinfer_semantic_bucket`.
- Update `classify_workload_readiness` to:
  - mark simple cases as ready with
    `flashinfer_pytorch_compatible_reference`, and
  - mark runtime cases as `runtime_blocked` with `flashinfer_specific`
    and specific reason codes.

### acceptance_criteria

- Simple FlashInfer workloads route to `ready`.
- Runtime-dependent workloads route to `runtime_blocked` with reason bucketing.
- Runtime blocker reports include `flashinfer_runtime_dependency`.

## Task 2: Add semantic coverage tests

In `tests/sol_execbench/test_dataset_inventory_readiness.py`, ensure:

- `test_readiness_classifies_flashinfer_semantic_buckets` covers all known
  simple and runtime workload names.
- Unknown runtime-dependent FlashInfer references still map to
  `flashinfer_runtime_unknown`.

### acceptance_criteria

- Regression tests verify all simple and runtime buckets.
- Tests confirm PyTorch-compatible reason and runtime blocker reason alignment.

## Done Definition

- `FLASHINFER` readiness behavior is split and deterministic.
- 26 FlashInfer-Bench problems are represented by explicit semantic branches.
- No category-wide string-only blocker remains.
