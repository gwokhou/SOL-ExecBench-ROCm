# Phase 161 Plan

## Goal

Classify partial profiler-backed RDNA4 timing targets by failure mode and
closure decision so profiler closure work does not retry correctness/runtime
failures as profiler lifecycle failures.

## Tasks

1. Add a partial failure classification script.
   - Read the current profiler coverage report.
   - Select `partial_profiler_backed` targets by default.
   - Extract trace status counts, profiled/expected workloads, evidence path,
     kernel rows, and failure reason.

2. Assign failure modes.
   - `invalid_reference_only`
   - `runtime_error_only`
   - `mixed_correctness_runtime`
   - `incomplete_workload_coverage`
   - `profiler_evidence_gap`

3. Assign closure decisions.
   - `blocked_on_correctness`
   - `blocked_on_runtime`
   - `blocked_on_mixed_failures`
   - `complete_missing_workload_slices`
   - `inspect_profiler_evidence_gap`

4. Emit artifacts.
   - JSON ledger.
   - Markdown summary.
   - Problem-id lists by closure decision.

5. Run real classification.
   - Generate `out/rdna4-profiler-partial-failure-classification-20260608`.
   - Produce a focused `L1/076_batched_expert_forward` deep-dive row.
