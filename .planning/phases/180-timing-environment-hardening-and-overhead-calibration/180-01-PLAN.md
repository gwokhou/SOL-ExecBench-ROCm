# Plan 180-01: GPU Device Isolation and Strict Isolation Mode

## Objective

Add GPU device isolation validation and a `--strict-isolation` abort mode to the timing
isolation module and profiler timing batch script, closing two of the three remaining
timing precision gaps.

## Tasks

### Task 1: Add `validate_gpu_device_isolation()` to timing_isolation.py

Add a new public function that:

1. Reads `ROCR_VISIBLE_DEVICES` env var
2. Queries GPU count via `rocm-smi --showid` (subprocess, 5s timeout, graceful degradation)
3. Optionally sets `ROCR_VISIBLE_DEVICES` when `gpu_device` parameter provided
4. Returns a structured result dict with: `isolated` (bool), `gpu_count` (int),
   `rocr_visible_devices` (str|None), `warnings` (list[str])
5. Does NOT abort — caller decides based on strict mode

Schema version: `sol_execbench.gpu_device_isolation.v1`

Also update `collect_timing_environment_snapshot()` to include the isolation result.

### Task 2: Add `--strict-isolation` and `--gpu-device` flags to batch script

Add two new CLI arguments to `run_rdna4_profiler_timing_batch.py`:

- `--strict-isolation` (bool, default False): upgrades warnings to abort
- `--gpu-device` (int, optional): sets ROCR_VISIBLE_DEVICES for this process

Wire into `run_batch()` pre-flight audit section:
- Call `validate_gpu_device_isolation(gpu_device=...)`
- If `strict_isolation=True` and `isolated=False` → log error and `sys.exit(1)`
- If `strict_isolation=False` and `isolated=False` → log warning (current behavior)
- Pass `strict_isolation` through `parse_args` to `run_batch`

### Task 3: Wire strict-isolation into concurrent process detection

When `--strict-isolation` is active, upgrade existing pre-flight warnings to abort:

- `detect_concurrent_gpu_processes()` returns non-empty → abort
- `verify_clock_state_with_warning()` returns False → abort
- `validate_gpu_device_isolation()` returns isolated=False → abort

All aborts include clear error messages identifying which check failed.

### Task 4: Write tests

- Unit tests for `validate_gpu_device_isolation()` (mocked rocm-smi)
- Unit tests for `--strict-isolation` flag parsing
- Integration test: strict mode aborts on concurrent GPU processes
- Integration test: strict mode aborts on clock unlock
- Integration test: default mode warns but proceeds on all failures
