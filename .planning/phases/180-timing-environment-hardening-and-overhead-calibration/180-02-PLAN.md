# Plan 180-02: rocprofv3 Overhead Calibration Script and Integration

## Objective

Create a script to measure rocprofv3 instrumentation overhead and integrate the
calibration value into profiler-backed timing evidence, enabling absolute timing
accuracy claims.

## Tasks

### Task 1: Create overhead calibration script

Create `scripts/run_rdna4_profiler_overhead_calibration.py` that:

1. Runs a minimal HIP kernel (vector add, 1M elements) N times WITHOUT rocprofv3
   → collect baseline device-event timings (median)
2. Runs the same kernel N times WITH rocprofv3
   → collect profiler-backed timings (median)
3. Computes overhead = profiler_median_ms - baseline_median_ms
4. Writes calibration result as a versioned JSON sidecar:
   ```json
   {
     "schema_version": "sol_execbench.rocprofv3_overhead_calibration.v1",
     "generated_at": "...",
     "baseline_median_ms": 0.052,
     "profiler_median_ms": 0.073,
     "overhead_ms": 0.021,
     "iterations": 100,
     "warmup_runs": 10,
     "gpu_architecture": "gfx1200",
     "clock_locked": true
   }
   ```

The script integrates Phase 175/176 infrastructure:
- Acquires PID lock unconditionally
- Runs timing isolation pre-flight audit
- Uses `--strict-isolation` by default (overhead calibration is data-sensitive)
- Supports `--strict-isolation / --no-strict-isolation` flag
- Supports `--gpu-device` flag

### Task 2: Add profiler_overhead_ms to timing evidence

Extend `build_timing_evidence()` in `rocm_profiler.py` to accept an optional
`profiler_overhead_ms` parameter. When provided, the value is included in the
resulting timing evidence dict.

Extend `collect_rocprofv3_timing()` to accept an optional `calibration_path`
parameter. When provided, reads the calibration JSON and passes the overhead
value through to `build_timing_evidence()`.

### Task 3: Write tests

- Unit tests for calibration JSON schema validation
- Unit tests for `profiler_overhead_ms` integration in `build_timing_evidence()`
- Unit tests for `collect_rocprofv3_timing()` with calibration path
- Test that calibration file reading handles missing/invalid files gracefully
