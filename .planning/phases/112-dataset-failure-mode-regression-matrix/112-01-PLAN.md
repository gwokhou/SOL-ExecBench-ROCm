# Phase 112 Plan: Add Dataset Failure-Mode Regression Matrix

## Goal

Dataset runner failure modes should be easy to audit and protected by tests and
documentation.

## Tasks

1. Add a concise failure-mode matrix to `docs/analysis.md`.
2. Add a doc regression test that checks the documented matrix includes stale
   provenance, ready subsets, missing sidecars, rerun, timeout/nonzero, and
   no-trace outcomes.
3. Run focused documentation and dataset closure tests.

## Acceptance Criteria

- Documentation explains when reuse is allowed, blocked, or reported
  incomplete.
- Documentation states that policy/closure tests are CPU-safe and live ROCm
  timing collection remains environment-dependent.
- Focused tests pass.
