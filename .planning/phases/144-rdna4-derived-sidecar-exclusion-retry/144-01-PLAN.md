# Phase 144 Plan

## Goal

Retry the temporary RDNA4 derived sidecar exclusions through isolated,
memory-capped jobs and classify any remaining blockers.

## Tasks

1. Build retry target inventory.
   - Parse the Phase 140 long-tail exclusion sidecar.
   - Group excluded records by problem and reason.
   - Preserve the historical exclusion config as input evidence.

2. Run isolated retries.
   - Retry excluded problems without the long-tail exclusion config.
   - Use `systemd-run --user` with `MemoryMax` and `MemorySwapMax=0`.
   - Persist per-problem logs and status JSONL.
   - Continue on failure so one OOM does not stop classification for the rest.

3. Audit generated sidecars.
   - Count AMD SOL v2 and SOLAR derivation sidecars for the previously excluded
     workload/problem set.
   - Classify retry outcomes as recovered, still memory-blocked, trace-status
     blocked, or tooling-blocked.

4. Close Phase 144.
   - Write summary and verification artifacts.
   - Update v1.31 requirements and route to Phase 145.

## Verification

- Retry status JSONL exists and records each attempted problem.
- Retry log exists and captures per-problem command output.
- A machine-readable Phase 144 summary records recovered vs blocked exclusions.
- Focused tests for touched code pass if implementation code changes are made.
