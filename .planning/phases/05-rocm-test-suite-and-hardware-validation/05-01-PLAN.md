# Plan 05-01: ROCm Pytest Hardware Markers

## Objective

Replace CUDA SM / Blackwell marker behavior with ROCm-aware marker detection and
skip reasons.

## Tasks

1. Update `tests/conftest.py` to detect ROCm through PyTorch's HIP-enabled
   `torch.cuda` compatibility API.
2. Register `requires_rocm`, `requires_rdna4`, `requires_cdna3`, and legacy
   `requires_cutile` marker descriptions.
3. Skip `requires_rocm` when no ROCm GPU is visible.
4. Skip `requires_rdna4` unless the detected AMD gfx architecture is RDNA 4.
5. Skip `requires_cdna3` unless the detected AMD gfx architecture is CDNA 3.
6. Treat `requires_cutile` as a legacy NVIDIA-only marker that is skipped in this
   ROCm-only port.
7. Update `pyproject.toml` marker descriptions to ROCm/HIP wording.

## Verification

- Focused pytest audit verifies marker registration and skip reason wording.
- `uv run --no-sync pytest ...` does not report unknown marker warnings.
