# Phase 128 Plan: MI300X Evidence Contract and Validation Handoff

**Created:** 2026-06-04
**Status:** Ready for execution
**Requirements:** MI300X-EVID-01, MI300X-EVID-02, MI300X-EVID-03,
MI300X-EVID-04

## Goal

Make the future MI300X/gfx942 validation handoff complete enough to execute and
audit, without performing hardware validation on the current host.

## Tasks

1. Expand diagnostics evidence constants and blockers.
   - Ensure required artifacts cover pytest log, dataset summary, environment
     report, clock-lock evidence, per-problem traces, timing evidence,
     AMD-native score report, and quantization status.
   - Preserve strict MI300X identity, `gfx94*`, ROCm version, clocks, and full
     suite blockers.

2. Document result classification.
   - Add expected skip, missing tool, functional failure, timing instability,
     missing evidence, FP8, and NVFP4/MXFP4 categories.
   - Keep these categories descriptive; do not treat them as validation.

3. Update handoff docs.
   - Update `.planning/milestones/MI300X-VALIDATION-HANDOFF.md`.
   - Update `.planning/milestones/CDNA3-VALIDATION-HANDOFF.md` if needed.
   - Update `docs/internal/mi300x_validation_readiness.md` and
     `docs/internal/cdna3_validation_readiness.md` if needed.

4. Strengthen tests.
   - Update diagnostics tests to assert expanded artifacts and blockers.
   - Update support-doc tests to assert command/evidence/failure categories.
