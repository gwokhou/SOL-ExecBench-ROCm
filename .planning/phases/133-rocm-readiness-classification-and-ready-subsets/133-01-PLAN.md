# Phase 133 Plan: ROCm Readiness Classification and Ready Subsets

## Goal

Implement CPU-safe static readiness classification for migrated workloads and
produce deterministic ready-subset/blocker sidecars without making validation,
paper-parity, or score-authority claims.

## Tasks

1. Extend `src/sol_execbench/core/dataset/readiness.py`.
   - Add Phase 133 readiness classes:
     `pytorch_compatible`, `rocm_port_needed`, `flashinfer_specific`,
     `nvfp4_blackwell_specific`, `unsupported`, and
     `blocked_missing_evidence`.
   - Add deterministic blocker records covering CUDA kernel dependencies,
     FlashInfer runtime assumptions, low-precision format dependencies, missing
     blobs/evidence, and unsupported dtypes.
   - Preserve legacy `status` values where current consumers depend on them.

2. Extend `src/sol_execbench/core/dataset/ready_subset.py`.
   - Preserve denominator and exclusion details in the ready-subset sidecar.
   - Include workload identity and closure input refs for included workloads.
   - Keep all validation and score-authority claim-boundary fields false.

3. Update package exports if new public models/functions are added.

4. Add CPU-safe tests.
   - Synthetic fixtures for each new readiness class.
   - Fixture coverage using Phase 132 migration outputs.
   - Guardrails proving ready subsets do not imply hardware validation, paper
     parity, or score authority.
