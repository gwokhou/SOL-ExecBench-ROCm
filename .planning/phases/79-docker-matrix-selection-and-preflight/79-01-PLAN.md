<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest file to Python selector | Checked-in JSON is parsed into strict Matrix Target policy. |
| user Target id to selector | Untrusted command-line Target id may attempt arbitrary image/tag selection. |
| host preflight observations to Matrix evidence | Host/device/Docker observations become diagnostic compatibility evidence. |
| diagnostic evidence to benchmark execution | Preflight decisions must not escalate into benchmark authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-79-01 | Tampering | `docker/rocm-targets.json` | mitigate | Strict manifest model with `extra=forbid`; tests assert declared 7.0.x/7.1.x/7.2.x entries and default id. |
| T-79-02 | Tampering | `select_docker_target` | mitigate | Reject unknown Target ids unless explicit unsafe/untested override is supplied; override emits `not_tested` only. |
| T-79-03 | Spoofing | preflight Matrix Entry | mitigate | Use Phase 78 `MatrixClaimBoundary` with all authority fields false and no clean validation flags for preflight failures. |
| T-79-04 | Repudiation | digest evidence | mitigate | Record requested repo/tag and nullable digest explicitly; digest absence is non-blocking and visible. |
| T-79-05 | Elevation of privilege | Docker Target selection | mitigate | This plan constructs build args only; it does not execute Docker, add privileges, or mount host paths. |
| T-79-SC | Tampering | package installs | accept | No new package installs; use Python stdlib, existing Pydantic, and existing Phase 78 models only. |
</threat_model>
