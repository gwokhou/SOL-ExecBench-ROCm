# Plan 132-01: Local Dataset Migration Pipeline

## Objective

Implement deterministic, local-only conversion tooling for user-downloaded
SOL-ExecBench and FlashInfer Trace inputs into the repository's ROCm
runner-compatible local problem layout, with manifests that preserve
provenance, license boundaries, checksums, artifact refs, and blocker states.

## Tasks

1. Add importable migration models and helpers in
   `src/sol_execbench/core/dataset/migration.py`.
   - Define schema versioned manifest, artifact, source, license-boundary,
     checksum, and blocker models.
   - Implement deterministic file checksum and manifest checksum behavior.
   - Implement safe relative path handling under the source root and output
     root.

2. Implement SOL-ExecBench local migration.
   - Accept source root, output root, optional categories, source revision,
     and created-at override.
   - Copy canonical `definition.json`, `workload.jsonl`, optional
     `config.json`, optional `reference.py`, and solution files into
     category/problem directories.
   - Record blocker states for missing required files, missing solutions, and
     missing safetensors blobs referenced by workload rows.

3. Implement FlashInfer Trace local migration.
   - Accept a source root with trace definitions/workloads/solutions/traces in
     common local layouts.
   - Normalize discovered problem directories into the same local benchmark
     problem layout.
   - Record blocker states for missing definitions, workloads, solutions,
     traces, and safetensors blobs.

4. Add CLI commands under the existing `sol-execbench` dispatch.
   - `sol-execbench dataset migrate-sol SOURCE_ROOT OUTPUT_ROOT ...`
   - `sol-execbench dataset migrate-flashinfer SOURCE_ROOT OUTPUT_ROOT ...`
   - Support `--source-revision`, `--manifest`, `--category`, and `--json`.

5. Add CPU-safe synthetic fixture tests.
   - Verify SOL migration output layout and deterministic manifest behavior.
   - Verify FlashInfer migration records missing traces/solutions and copies
     available local files.
   - Verify safetensors references become explicit blockers when local blobs
     are absent.
   - Verify CLI commands run without network or external datasets.

6. Verify and close the phase.
   - Run targeted pytest and Ruff.
   - Write `132-SUMMARY.md` and `132-VERIFICATION.md`.
   - Update requirements/roadmap/state if required by GSD status conventions.
   - Commit atomically with DCO sign-off.

## Acceptance Criteria

- DATA-MIG-01: Synthetic SOL-ExecBench rows migrate into local benchmark layout.
- DATA-MIG-02: Synthetic FlashInfer Trace inputs normalize into local benchmark
  layout with trace and solution refs.
- DATA-MIG-03: Manifest records deterministic source IDs, revisions,
  checksums, generated artifact refs, license-boundary metadata, and manifest
  checksum.
- DATA-MIG-04: Missing blobs, safetensors refs, traces, and solution records
  produce explicit blockers or warnings instead of silent omission.
