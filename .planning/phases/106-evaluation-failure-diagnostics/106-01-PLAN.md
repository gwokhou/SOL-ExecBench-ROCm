# Phase 106 Plan 01: Bounded No-Trace Diagnostics

## Goal

Persist bounded diagnostics for evaluator outcomes that produce no parseable
trace JSONL, and point CLI users to those diagnostics without changing
canonical trace output.

## Tasks

1. Add CLI helper functions in `src/sol_execbench/cli/main.py` to derive a
   deterministic no-trace diagnostics sidecar path, bound stdout/stderr tails,
   and write a diagnostic-only JSON payload.
2. Wire the helper into both evaluator failure branches:
   - nonzero return code with empty stdout
   - no parseable traces after stdout parsing
3. Keep normal trace writing, `--json`, environment/profile/static sidecars,
   and canonical trace schemas unchanged.
4. Add tests in `tests/sol_execbench/test_cli_environment_snapshot.py` for
   path derivation, bounded payloads, no-output failure payloads, noisy stdout,
   and zero-trace behavior.
5. Run the focused CLI sidecar tests and a public contract guardrail test that
   protects canonical trace key space.

## Verification

- `uv run pytest tests/sol_execbench/test_public_contract_guardrails.py -q`
