## Integration Checker Findings

### Blockers

1. **AMD SOL/SOLAR evidence is not wired past Phase 89**

   Phase 89 accepts `amd_sol_report` and `solar_derivation` inputs, but
   claim-upgrade and trust-summary do not accept AMD SOL/SOLAR inputs or expose
   direct source refs/checksums for them.

   Affected requirements: CLAIM-02, TRUST-01, TRUST-03, partly DOCS-01.

2. **DOCS-04 promises a consistent example shape, but no consistent fixture exists**

   `docs/v1_20_evidence_quality_guide.md` says fixtures show consistent,
   contradictory, noisy, and claim-blocked report shapes. The fixture directory
   does not include a consistent fixture, and the docs test does not require
   one.

   Affected requirement: DOCS-04.

### Warning

3. **No automated full cross-script E2E chain test**

   Each script is tested independently, but no regression runs:

   `report_consistency.py -> report_evaluation_stability.py -> report_claim_upgrade.py -> report_trust_summary.py`

   using generated outputs as downstream inputs.

   Affected requirements: DOCS-03, TRUST-01, CLAIM-02.
