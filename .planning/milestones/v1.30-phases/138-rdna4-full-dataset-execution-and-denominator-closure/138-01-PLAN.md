# Phase 138 Plan

## Goal

Execute the RDNA4 dataset validation path with complete problem/workload
denominator accounting and no silent omissions.

## Preconditions

- Local SOL ExecBench benchmark assets are present under
  `data/SOL-ExecBench/benchmark` or a user-provided equivalent path.
- `uv run` can see `/dev/kfd` and `/dev/dri`.
- PyTorch ROCm in the `uv run` environment reports HIP support, at least one
  visible device, and `gfx1200`.
- Optional long-tail exclusion config is explicit and archived with evidence.

## Execution Command Template

```bash
UV_CACHE_DIR=/tmp/uv-cache uv run scripts/run_dataset.py \
  data/SOL-ExecBench/benchmark \
  --output out/rdna4-full-dataset \
  --execution-closure out/rdna4-full-dataset/execution_closure.json \
  --gpu-architecture gfx1200 \
  --timeout-policy record \
  --workload-shard-size 1 \
  --rerun
```

If a long-tail exclusion config is approved, add:

```bash
--long-tail-exclusions .planning/phases/138-rdna4-full-dataset-execution-and-denominator-closure/138-long-tail-exclusions.json
```

## Tasks

1. Verify dataset availability and denominator discovery.
   - Confirm category directories and problem/workload counts.
   - Record dataset manifest/checksum refs when present.

2. Verify RDNA4 command-environment access.
   - Run the PyTorch ROCm probe from Phase 137 inside the exact command
     environment.
   - Stop if `uv run` cannot see device nodes or `gfx1200`.

3. Execute the full dataset run with checkpoint-aware polling.
   - Poll healthy long-running jobs instead of terminating by elapsed time.
   - Preserve traces, summaries, closure, CLI logs, and partial artifacts.
   - Use timeout traces rather than silent no-trace omissions.

4. Classify every denominator entry.
   - `attempted_passed`
   - `attempted_failed`
   - `excluded_long_tail`
   - `not_attempted`
   - `filtered`
   - `missing_trace`
   - `derived_evidence_missing`

5. Produce Phase 138 closure summary.
   - Summarize pass/fail/timeout/skip/exclusion counts.
   - Record follow-up issues for unaccepted correctness, runtime, dependency,
     or timeout failures.

environment accessed the RDNA4 `gfx1200` device.

The phase completed the bounded RDNA4 ready-subset execution. Historical
