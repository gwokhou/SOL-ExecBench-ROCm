# Phase 139 Plan

## Goal

Capture RDNA4 timing context for the Phase 138 ready-subset run with explicit
environment, clock-lock, profiler, and stability evidence.

## Tasks

1. Capture RDNA4 environment evidence.
   - Run `sol-execbench doctor --json`.
   - Capture `rocm-smi` product, driver, clock, power, temperature, and
     performance-level output.
   - Capture `rocprofv3 --version`.
   - Store generated evidence under `out/rdna4-timing-evidence/environment`.

2. Capture clock-lock status.
   - Record current DPM/performance state before timing.
   - Attempt a non-destructive lock-status verification first.
   - If lock setting requires privileges or is unsupported on this device,
     record an explicit unlocked-clock blocker and keep timing
     non-authoritative.

3. Collect `rocprofv3` timing evidence.
   - Use `scripts/run_dataset.py --phase timing` against the Phase 138 output
     and ready subset.
   - Write per-problem timing evidence under
     `out/rdna4-timing-evidence/timing`.
   - Keep the run serial and poll long-running profiler work at roughly
     10-minute intervals.

4. Generate timing stability reports.
   - Run `scripts/report_evaluation_stability.py` over collected timing
     evidence.
   - Write JSON and Markdown reports under
     `out/rdna4-timing-evidence/stability`.
   - Classify missing timing, unlocked clock, profiler fallback/failure, and
     dispersion/overhead risk.

5. Close Phase 139.
   - Record artifact refs, counts, blockers, and claim boundaries in
     `139-SUMMARY.md` and `139-VERIFICATION.md`.
   - Update `REQUIREMENTS.md`, `ROADMAP.md`, and `STATE.md`.

## Verification

- Environment sidecars exist and identify RDNA4 `gfx1200`.
- Clock-lock status is explicit.
- `rocprofv3` timing evidence exists or every missing profiler artifact is
  classified with a concrete blocker.
- Stability report exists and preserves non-authoritative timing boundaries
  when clocks are unlocked or profiler evidence is incomplete.
