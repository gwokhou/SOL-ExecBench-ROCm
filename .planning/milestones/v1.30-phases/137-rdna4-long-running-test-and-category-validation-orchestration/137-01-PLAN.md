# Phase 137 Plan

## Goal

Produce a resumable RDNA4 validation orchestration path and collect adapted
pytest/category evidence on visible `gfx1200` hardware, while preserving clear
classification for skips, failures, timeouts, dependency gaps, and known
boundaries.

## Tasks

1. Create the RDNA4 long-running validation runbook.
   - Define preflight commands for `rocminfo`, `rocm-smi`, `lspci`, `hipcc`,
     PyTorch ROCm visibility, and marker selection.
   - Define polling cadence and checkpoint expectations for many-hour runs.
   - Define resume behavior for pytest, category examples, and dataset handoff.

2. Add machine-readable Phase 137 evidence artifacts.
   - Record command lines, host/GPU observations, run status, and category
     classifications.
   - Classify failures as correctness, compile, runtime, dependency, timeout,
     expected skip, or accepted known gap.
   - Keep category evidence bounded to Phase 137 and prevent benchmark-grade
     overclaiming.

3. Execute RDNA4 adapted pytest evidence where feasible.
   - Run `uv run pytest -m requires_rdna4 -q -rs` with polling rather than
     premature termination.
   - Preserve output in phase artifacts and summarize pass/skip/failure counts.

4. Execute focused category validation where feasible.
   - Cover HIP/C++, Triton ROCm, hipBLAS, MIOpen, CK, and rocWMMA via existing
     examples/tests.
   - Record accepted dependency or hardware boundaries for unavailable
     categories.

5. Add guardrails/tests for category claim boundaries.
   - Prevent Phase 137 category readiness from implying Phase 138 full dataset
     validation, Phase 139 timing authority, or Phase 141 public claim closure.

## Verification

- `uv run pytest -m requires_rdna4 -q -rs` when RDNA4 hardware is visible.
- Focused category pytest command covering examples and ROCm library tests.
- CPU-safe guardrail tests for the Phase 137 evidence/runbook artifacts.
- `uv run --with ruff ruff check` for changed Python files.
- `git diff --check`.

## Non-Goals

- Full dataset denominator execution.
- Benchmark-grade timing or clock-lock authority.
- Public claim upgrade.
- CDNA3/MI300X or CDNA4 validation.
