# Requirements: SOL ExecBench ROCm Port v1.12

**Defined:** 2026-05-25
**Milestone:** v1.12 Evaluator Contract Metadata and Boundary Guardrails
**Core Value:** Evaluate LLM-generated GPU kernels correctly and reproducibly on
AMD ROCm hardware while preserving the benchmark semantics and rigor of SOL
ExecBench.

## v1.12 Requirements

### Evaluator Contract Metadata

- [x] **CONTRACT-01**: Maintainer can emit a GPU-free evaluator compatibility
  contract from the primary CLI using `sol-execbench contract --json` without a
  problem directory or GPU evaluation.
- [x] **CONTRACT-02**: Contract metadata records stable schema and contract
  versions, capability tokens, trace field groups, correctness fields, timing
  fields, scoring fields, evaluation statuses, failure categories, baseline
  export fields, compatibility metadata fields, and source-boundary claims.
- [x] **CONTRACT-03**: Contract metadata is available through the data package
  API without mutating canonical `Trace`, `Definition`, `Workload`, or
  `Solution` public schemas.

### Ownership Boundary Guardrails

- [x] **BOUNDARY-01**: SOL remains the owner of correctness, timing, scoring,
  trace semantics, ROCm execution, and baseline/scoring contract truth.
- [x] **BOUNDARY-02**: HIP-side or playground consumers are represented as
  external consumers of SOL-owned JSON contracts and do not introduce
  `hip_agent` runtime semantics into SOL source.
- [x] **BOUNDARY-03**: Measured baseline registry metadata uses a SOL-owned
  schema namespace, `sol_execbench.measured_baseline_registry.v1`.
- [x] **BOUNDARY-04**: Guardrail tests prevent contract-only metadata fields
  from leaking into canonical trace JSONL.

### Shipping Evidence

- [x] **SHIP-01**: The work is merged into `gwokhou/SOL-ExecBench-ROCm:main`
  through PR #1, not proposed to NVIDIA upstream.
- [x] **SHIP-02**: The feature work is represented by a DCO-signed commit,
  `5d4892d #17 - Expose SOL evaluator contract metadata`.
- [x] **SHIP-03**: Targeted verification passed:
  `37 passed, 1 skipped` for contract, boundary, and public-contract guardrail
  tests.

## Out of Scope

| Feature | Reason |
