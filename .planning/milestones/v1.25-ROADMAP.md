# Roadmap Archive: v1.25 Engineering Prerelease

**Shipped:** 2026-06-01

## Goal

Turn the v1.24 ROCm port state into an engineering prerelease /
release-candidate package that external users can install, validate, and
interpret without overstated research or hardware claims.

## Completed Phases

- Phase 114: Release-Candidate Validation
- Phase 115: Support Matrix Boundaries
- Phase 116: Claim Boundary Guardrails
- Phase 117: First-Run User Path
- Phase 118: Release Candidate Materials

## Outcomes

- Maintainers can run bounded release-candidate validation across CPU-safe
  checks, optional ROCm/Docker smoke, and bounded dataset-slice paths.
- Public support docs distinguish RDNA 4 prerelease evidence, Docker/container
  ROCm user-space evidence, deferred MI300X/CDNA3 full-suite validation, and
  unavailable CDNA4 validation.
- Release docs classify canonical, diagnostic-only, provisional, deferred, and
  unavailable evidence and prevent paper-parity, upstream SOLAR, leaderboard,
  hard-sandbox, and hardware overclaims.
- First-run docs guide users through installation, minimal trace generation,
  trace interpretation, no-trace diagnostics, and PyTorch ROCm compatibility
  wording.
- Release-candidate materials include a maintainer checklist, release notes,
  README navigation, and CPU-safe guardrail tests.
