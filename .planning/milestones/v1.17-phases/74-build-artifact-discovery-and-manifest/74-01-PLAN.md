<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller-supplied build root -> discovery | Build root may contain symlinks or unrelated files. |
| Build staging directory -> durable evidence directory | Artifacts must be copied without escaping source boundary or corrupting manifest paths. |
| Diagnostic manifest -> benchmark truth surfaces | Static artifact metadata must remain non-authoritative. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-01 | Spoofing | Artifact source boundary | mitigate | Require explicit build root and primary artifact; no environment-derived search roots. |
| T-74-02 | Tampering | Persisted artifact metadata | mitigate | Hash persisted bytes and record persisted file size. |
| T-74-03 | Repudiation | Manifest provenance | mitigate | Record source and persisted paths plus kind/provenance references. |
| T-74-04 | Information Disclosure | Path traversal / broad scans | mitigate | Resolve paths, enforce root containment, reject symlink escapes, and test outside artifacts. |
| T-74-05 | Denial of Service | Optional evidence absence | mitigate | Return unavailable/unsupported sidecars instead of failing benchmark behavior. |
| T-74-06 | Elevation of Privilege | Extractor/tool execution | mitigate | No subprocess, tool lookup, CLI, or extractor execution in Phase 74. |
| T-74-SC | Supply Chain | Package installs | mitigate | No new dependencies or package-manager commands. |
</threat_model>
