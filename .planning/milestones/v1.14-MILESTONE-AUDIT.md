# Milestone Audit: v1.14 Optional rocprofv3 Profiling Evidence

**Audited:** 2026-05-25
**Status:** PASSED

## Scope

v1.14 set out to add opt-in `rocprofv3` profiling artifacts as diagnostic
benchmark evidence while preserving default execution, canonical trace JSONL,
correctness, timing, scoring, and evaluator contract version `1.0`.

## Requirement Coverage

| Area | Requirements | Status |
|------|--------------|--------|
| Profiling invocation | PROF-01, PROF-02, PROF-03 | Complete |
| Artifact capture | ART-01, ART-02, ART-03 | Complete |
| Reporting and docs | REPORT-01, REPORT-02, REPORT-03 | Complete |

All 9 v1.14 requirements are satisfied.

## Delivered Evidence

- Phase 61: `--profile rocprofv3`, profile command construction, command
  provenance, availability checks, and normal-execution fallback.
- Phase 62: stable profile artifact directory, profile metadata sidecar,
  `rocpd`/CSV artifact registration, return code, skipped reason, failed
  reason, and stdout/stderr tails.
- Phase 63: optional `profiling.evidence.v1` capability, ROCm usage docs,
  timing-semantics documentation, and public contract guardrails.

## Compatibility Audit

- Profiling is disabled by default through `--profile none`.
- The existing trace JSONL model and top-level keys are unchanged.
- Evaluator contract schema remains `sol_execbench.evaluator_contract.v1`.
- Evaluator contract version remains `1.0`.
- `profiling.evidence.v1` is an optional capability only.
- Missing or failed `rocprofv3` collection records diagnostic metadata and then
  runs the normal benchmark path.
- Profile sidecars are marked `diagnostic_only: true` and
  `score_authority: false`.
