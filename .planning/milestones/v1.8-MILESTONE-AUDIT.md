- Phase 36 dependency diagnostics identify missing library headers and link
  libraries for all native library categories introduced or promoted later.
- Phases 37-39 each add a public native example, solution metadata, embedded
  source consistency coverage, native staging coverage, and RDNA 4 E2E
  registration for their library category.
- Phase 40 aligns README, ROCm setup docs, and library readiness docs so support
  status is tied to runnable example scope rather than broad ecosystem claims.
- Former NVIDIA category directories remain PyTorch compatibility examples
  unless they contain native ROCm library solutions.
- CDNA 3 and CDNA 4 validation remain explicitly deferred, matching the user's
  v1.8 hardware-scope decision.

replacement ecosystem by adding diagnostics, public native examples, tests, and

scoped to RDNA 4 and deferring CDNA 3/CDNA 4 hardware evidence.
