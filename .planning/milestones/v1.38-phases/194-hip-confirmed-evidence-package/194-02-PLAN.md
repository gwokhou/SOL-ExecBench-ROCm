# Phase 194-02 Plan: Official Score Emission CLI + Blocker-Removal (GATE-03)

**Requirement:** GATE-03
**Wave:** 1 (independent of 194-01; consumes the 193 gate + coverage modules)
**Files created:** `src/sol_execbench/cli/commands/official_score.py`,
`tests/sol_execbench/cli/commands/test_official_score_cli.py`
**Files modified:** `src/sol_execbench/cli/commands/root.py` (register command);
`src/sol_execbench/core/scoring/amd_score/models.py` or
`src/sol_execbench/core/scoring/amd_score/__init__.py` (add a `from_dict` loader
if none exists)

## Objective

Provide a SOL emission surface so HIP can obtain `official_score_evidence.v1`
from SOL-provided benchmark evidence and remove `missing_score` /
`missing_baseline` / `placeholder_baseline` blockers for valid runs. Mirrors the
agent_feedback / profile_summary sidecar-producer pattern: input SOL artifacts →
output confirmed-evidence artifact.

## Decisions Covered

- D-09: Add `sol-execbench official-score` CLI emitting `official_score_evidence.v1`
  JSON from inputs.
- D-10: `--aggregation-policy` is a REQUIRED flag (resolves the 193 STAGING note's
  unresolved aggregation-policy decision without adding it to
  `AmdNativeSuiteReport`).
- D-11: The CLI builds the coverage report from the measured baseline registry
  plus `--current-run-env` flags, then passes it as the gate's `coverage_report`.
- D-12: A round-trip test proves valid runs emit no `missing_score` /
  `missing_baseline` / `placeholder_baseline` blockers and invalid runs keep
  precise blocker codes.

## Implementation Steps

1. Read before editing:
   - `src/sol_execbench/core/scoring/official_score.py` (gate API +
     `build_official_score_suite_evidence`, `coverage_report` precondition).
   - `src/sol_execbench/core/scoring/amd_score/models.py`
     (`AmdNativeScore`, `AmdNativeSuiteReport` — check for an existing
     `from_dict`; if absent, add one).
   - `src/sol_execbench/core/evidence/baseline_coverage.py`
     (`validate_baseline_coverage`, `CurrentRunEnvironment`).
   - `src/sol_execbench/core/evidence/baseline_export.py` (registry shape).
   - `src/sol_execbench/cli/commands/metadata.py` and
     `src/sol_execbench/cli/commands/baseline.py` (CLI patterns), and
     `src/sol_execbench/cli/commands/root.py` (`_subcommand_for` dispatch).
2. If `AmdNativeSuiteReport` / `AmdNativeScore` lack a `from_dict` loader, add a
   small inverse loader (`amd_native_suite_report_from_dict`) next to the model
   (mirror `scoring_baseline_artifact_from_dict` in
   `src/sol_execbench/core/scoring/baseline_artifact.py`). Reuse it in the CLI.
3. Create `src/sol_execbench/cli/commands/official_score.py` with a click command
   `official-score` (GPU-free). Options:
   - `--amd-native-score <path>` (required): an `amd_native_score.v1` report JSON.
   - `--measured-registry <path>` (required): a measured baseline registry JSON
     (the `baseline_export.py` output shape).
   - `--aggregation-policy <text>` (required): the aggregation policy string.
   - `--current-run-env-hardware`, `--current-run-env-rocm`,
     `--current-run-env-target`, `--current-run-env-timing-policy` (optional
     strings; omitted fields are treated as "do not compare").
   - `--output <path>` (optional): write the evidence JSON to a file; if omitted,
     echo to stdout.
   - Behavior: load the amd-native report → `AmdNativeScore` iterable; load the
     measured registry dict; build `CurrentRunEnvironment` from the env flags;
     `validate_baseline_coverage(registry, current_run_environment=env)` →
     coverage report; `build_official_score_suite_evidence(scores,
     aggregation_policy=policy, coverage_report=report)`; write
     `suite.to_dict()` as `official_score_evidence.v1` JSON.
4. Register the command in `src/sol_execbench/cli/commands/root.py`
   `_subcommand_for` (add `if name == "official-score": ...` returning
   `cli_official_score._official_score_cli`). Match the existing dispatch style.
5. Create `tests/sol_execbench/cli/commands/test_official_score_cli.py`:
   - **Valid run round-trip:** a fixture amd-native report backed by
     `scoring_baseline` + a confirmed measured-registry coverage → emitted
     evidence has `score_authority`, no `missing_score` / `missing_baseline` /
     `placeholder_baseline` blockers (D-12, GATE-03 valid case).
   - **Invalid run — missing baseline:** report with `baseline_source="missing"`
     → emitted evidence keeps `missing_baseline`.
   - **Invalid run — placeholder baseline:** report with
     `baseline_source="reference_latency"` → keeps `placeholder_baseline`.
   - **Coverage failure:** a mismatched measured registry → emitted evidence
     keeps `baseline_coverage_failed` + propagated codes, score null.
   - **Missing aggregation policy:** CLI refuses (required flag).
   - Use `CliRunner` (see `test_baseline.py` / `test_baseline_export.py` for the
     pattern); build the amd-native report JSON with
     `build_amd_native_suite_report` helpers or hand-authored payloads loaded via
     the new `from_dict`.
