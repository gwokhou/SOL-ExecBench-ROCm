# Phase 194-04 Plan: Confirmed-Evidence Documentation (GATE-02 docs + GATE-01 wording)

**Requirement:** GATE-02 (docs), GATE-01 (authority wording)
**Wave:** 2 (**Depends on:** 194-01 + 194-02 + 194-03 — documents what they ship)
**Files modified:** `docs/user/EVALUATOR-CONTRACT.md`
**Files created:** `docs/user/confirmed_evidence.md`

## Objective

Publish HIP-facing documentation that explains which SOL artifacts HIP must
consume for confirmed pass/fail decisions, which artifacts remain diagnostic-only,
the stable blocker reason-code vocabulary, and the six fixture cases — so HIP can
make cutover-gate decisions from SOL-provided evidence without over-reading
diagnostic sidecars.

## Decisions Covered

- D-13: Extend `docs/user/EVALUATOR-CONTRACT.md` — `## Official Score Evidence`,
  `## Capabilities`, and `## Ownership Boundary` sections with the confirmed
  evidence surface, blocker vocabulary, and confirmed boundaries.
- D-14: Add `docs/user/confirmed_evidence.md` — a HIP-facing consume/diagnostic-only
  table.
- D-15: Document the six fixture cases in `docs/user/confirmed_evidence.md`.
- D-16: Reaffirm authority wording — diagnostic sidecars cannot provide confirmed
  pass/fail authority.

## Implementation Steps

1. Read before editing:
   - `docs/user/EVALUATOR-CONTRACT.md` (current structure: Canonical Authority,
     Capabilities, Official Score Evidence, Feedback Sidecars, Ownership
     Boundary).
   - `docs/user/profile_summary_sidecar.md` and `docs/user/agent_feedback_sidecar.md`
     (diagnostic-sidecar doc conventions to mirror).
   - The shipped code from 194-01/02/03 (contract capabilities, CLI, fixtures) so
     the docs match what was implemented.
2. Extend `docs/user/EVALUATOR-CONTRACT.md`:
   - `## Capabilities`: list `official_score.evidence` (confirmed) and
     `measured_baseline.coverage` (confirmed), and note the diagnostic sidecar
     capabilities remain `profile:diagnostic` / `optional`. (This also satisfies
     the stale-token guard for the new capability tokens added in 194-01.)
   - `## Official Score Evidence`: describe the `official_score_evidence.v1`
     artifact, the `sol-execbench official-score` emission CLI, the required
     `--aggregation-policy`, and the stable blocker reason-code vocabulary
     (`missing_score`, `missing_measured_latency`, `missing_baseline`,
     `placeholder_baseline`, `missing_sol_bound`, `missing_aggregation_policy`,
     `baseline_coverage_failed`).
   - `## Ownership Boundary`: state that `official_score` and `measured_baseline`
     are `confirmed` authority; diagnostic sidecars remain `diagnostic`.
3. Create `docs/user/confirmed_evidence.md`:
   - **HIP consume vs diagnostic-only table**: which SOL artifacts HIP MUST
     consume for confirmed pass/fail (`official_score_evidence.v1` +
     measured baseline coverage report) vs which remain diagnostic-only
     (`agent_feedback`, `profile_summary`, `decision` sidecars).
   - **Blocker removal (GATE-03)**: how valid SOL runs remove
     `missing_score`/`missing_baseline`/`placeholder_baseline` blockers and how
     invalid runs keep precise codes.
   - **The six fixture cases** (D-15): for each case in
     `tests/sol_execbench/fixtures/confirmed_evidence/`, document intent +
     expected blocker set + expected score authority.
   - **Authority wording (D-16)**: explicit statement that diagnostic sidecars
     cannot provide confirmed pass/fail, release-gate, cutover, paper-parity, or
     leaderboard authority; only official_score + measured baseline coverage can.
4. Ensure the contract stale-token guard tests still pass after doc edits (the
   new capability tokens appear in docs; no stale tokens introduced).
