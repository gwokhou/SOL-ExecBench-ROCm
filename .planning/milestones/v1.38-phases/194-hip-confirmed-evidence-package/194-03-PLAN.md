# Phase 194-03 Plan: HIP-Facing Confirmed-Evidence Fixtures (GATE-02)

**Requirement:** GATE-02
**Wave:** 2 (**Depends on:** 194-02 — bundle payloads mirror what the gate/CLI emit)
**Files created:** `tests/sol_execbench/fixtures/confirmed_evidence/*.bundle.json`
+ `*.case.json` (6 cases),
`tests/sol_execbench/core/evidence/test_confirmed_evidence_fixtures.py`

## Objective

Publish HIP-facing fixtures covering the six confirmed/missing/placeholder/
profiler-partial/diagnostic-only cases so HIP can validate its cutover-gate
consumption against stable, machine-verifiable SOL evidence bundles.

## Decisions Covered

- D-05: New fixture directory `tests/sol_execbench/fixtures/confirmed_evidence/`.
- D-06: One bundle JSON per case combining official_score_evidence + measured
  baseline registry summary + coverage summary (one coherent bundle per
  scenario).
- D-07: Six required cases (`confirmed-pass`, `missing-score`, `missing-baseline`,
  `placeholder-baseline`, `profiler-partial`, `diagnostic-only-sidecar`), each a
  `<case>.bundle.json` + `<case>.case.json` metadata file.
- D-08: A loader test loads each bundle and asserts its blocker reason-code set
  matches the GATE-03 expected set.

## Implementation Steps

1. Read before authoring fixtures:
   - `tests/sol_execbench/fixtures/profile_summary/` and
     `tests/sol_execbench/fixtures/agent_feedback/` (the `<state>.<artifact>.json`
     + `.case.json` convention — mirror it).
   - `src/sol_execbench/core/scoring/official_score.py`
     (`OfficialScoreSuiteEvidence.to_dict()` shape — the bundle's
     official_score_evidence section must match it).
   - `src/sol_execbench/core/evidence/baseline_coverage.py`
     (`BaselineCoverageReport.to_dict()` shape — the coverage summary section).
   - `src/sol_execbench/core/evidence/baseline_export.py` (registry shape).
2. Create `tests/sol_execbench/fixtures/confirmed_evidence/` with six bundles.
   Each `*.bundle.json` contains:
   - `case`: the case name.
   - `official_score_evidence`: an `OfficialScoreSuiteEvidence`-shaped payload
     (per case: confirmed-pass has non-null score + no blockers; the failure
     cases have null score + the precise blockers).
   - `measured_baseline_registry`: a registry summary (expected_workload_keys +
     a minimal entries list sufficient for the case).
   - `coverage_summary`: a `BaselineCoverageReport`-shaped summary (state_summary
     + all_confirmed + blocker_reason_codes).
   - `diagnostic_sidecars` (optional, for profiler-partial and
     diagnostic-only-sidecar cases): a reference to a profile_summary /
     agent_feedback fixture present, marked diagnostic-only.
   Each `*.case.json` metadata file records: `case`, `description`,
   `expected_blockers` (the exact blocker set HIP should observe), and
   `expected_score_authority` (bool).
   Case expected blockers:
   - `confirmed-pass`: `[]` (score authority true).
   - `missing-score`: `["missing_score"]`.
   - `missing-baseline`: `["missing_baseline"]`.
   - `placeholder-baseline`: `["placeholder_baseline"]`.
   - `profiler-partial`: official score blocked (`missing_baseline` or
     `baseline_coverage_failed`) AND a partial profile_summary present — asserts
     the diagnostic sidecar does NOT remove the blocker.
   - `diagnostic-only-sidecar`: only diagnostic sidecars present, no confirmed
     evidence → blockers remain (`missing_score`, `missing_baseline`).
3. Create `tests/sol_execbench/core/evidence/test_confirmed_evidence_fixtures.py`:
   - For each of the six bundles: load the `*.bundle.json` and `*.case.json`,
     assert `official_score_evidence.blocker_reason_codes` (suite-level) matches
     `expected_blockers`, and `official_score_evidence.score_authority` matches
     `expected_score_authority`.
   - For `profiler-partial` and `diagnostic-only-sidecar`: assert the diagnostic
     sidecar presence does not change the blocker set (diagnostic sidecars cannot
     remove confirmed-evidence blockers).
   - Assert every bundle's `official_score_evidence.schema_version` is
     `sol_execbench.official_score_evidence.v1`.
4. Ensure the contract stale-token guard
   (`test_active_source_and_fixture_text_avoid_stale_sidecar_capability_tokens`)
   still passes after adding the new fixture directory.
