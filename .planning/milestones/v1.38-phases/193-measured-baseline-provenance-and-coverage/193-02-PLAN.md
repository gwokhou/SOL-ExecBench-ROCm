# Phase 193-02 Plan: Five-State Baseline Coverage Validation (BASE-02)

**Requirement:** BASE-02
**Wave:** 1 (independent — no dependency on 193-01; reads the registry shape but
not the timestamp field)
**Files created:** `src/sol_execbench/core/evidence/baseline_coverage.py`,
`tests/sol_execbench/core/evidence/test_baseline_coverage.py`
**Files modified:** `src/sol_execbench/core/evidence/__init__.py` (exports)

## Objective

Implement a measured-baseline coverage validator that classifies each expected
workload's baseline into one of five stable states (`confirmed`, `missing`,
`stale`, `mismatched`, `placeholder`) with stable reason codes, comparing each
baseline entry's recorded provenance against an explicit current-run
environment. This is the core remaining work of Phase 193.

## Decisions Covered

- D-01: New module `src/sol_execbench/core/evidence/baseline_coverage.py` as a
  sibling of `baseline_export.py`.
- D-02: Five states — `confirmed`, `missing`, `stale`, `mismatched`,
  `placeholder` — each with a stable reason code.
- D-03: The validator accepts an explicit **current-run environment** parameter
  (hardware, ROCm version, target id, timing policy) and compares it against
  each baseline entry's recorded provenance to detect `mismatched`.
- D-04/D-17: `stale` is detected when the baseline entry's `trace_ref` file no
  longer exists (primary signal) OR provenance/generation is inconsistent with
  the current run.
- D-05: Report shape = per-workload rows `{workload_key, state, reason_code,
  detail}` plus suite-level state counts, mirroring
  `OfficialScoreSuiteEvidence.blocker_summary`.
- D-06: Per-state primary codes plus sub-codes so each distinct success-criteria
  #4 test case has its own code.
- D-07: snake_case `baseline_*` naming for the NEW codes.
- D-08: Coverage `missing` and `placeholder` states REUSE the existing
  official-score blocker literals `missing_baseline` and `placeholder_baseline`
  (no parallel namespace). NEW `stale`/`mismatched` codes use the `baseline_*`
  prefix.
- D-09: `confirmed` is a positive STATUS (passing state), not a blocker code.

## Reason-Code Scheme (locked)

| State | Primary reason code | Sub-code(s) | Detail |
|-------|---------------------|-------------|--------|
| `confirmed` | _(status only — no blocker)_ | — | entry covers workload, provenance matches current run |
| `missing` | `missing_baseline` | — | expected workload has no baseline entry |
| `placeholder` | `placeholder_baseline` | — | baseline entry is reference-latency-derived, not measured |
| `stale` | `baseline_stale` | `baseline_stale_trace` | `trace_ref` file missing / unreachable |
| `mismatched` | `baseline_mismatched` | `baseline_hardware_mismatch`, `baseline_timing_policy_mismatch` | provenance field differs from current-run env |

The coverage module MUST NOT import from `sol_execbench.core.scoring` (avoid
evidence→scoring layering). It defines the literal strings
`"missing_baseline"` / `"placeholder_baseline"` locally and documents them as
"matches official_score blockers". A consistency test pins the equality (see
Plan 193-03).

## Implementation Steps

1. Read before editing:
   - `src/sol_execbench/core/evidence/baseline_export.py` (registry payload
     shape: `expected_workload_keys`, per-entry `trace_ref`, `provenance`
     {hardware, rocm_version, sol_version, target_id, timing_policy}, `source`,
     `facts.reference_latency_ms`).
   - `src/sol_execbench/core/scoring/official_score.py` (the existing
     `MISSING_BASELINE_BLOCKER` / `PLACEHOLDER_BASELINE_BLOCKER` literal values
     and `_PLACEHOLDER_BASELINE_SOURCES` reference-latency patterns — mirror
     these for the placeholder classification).
   - `src/sol_execbench/core/evidence/__init__.py` (export style).
2. Create `src/sol_execbench/core/evidence/baseline_coverage.py` with:
   - State constants: `CONFIRMED`, `MISSING`, `STALE`, `MISMATCHED`,
     `PLACEHOLDER`.
   - Reason-code constants: `MISSING_BASELINE_CODE = "missing_baseline"`,
     `PLACEHOLDER_BASELINE_CODE = "placeholder_baseline"`,
     `BASELINE_STALE_CODE = "baseline_stale"`,
     `BASELINE_STALE_TRACE_CODE = "baseline_stale_trace"`,
     `BASELINE_MISMATCHED_CODE = "baseline_mismatched"`,
     `BASELINE_HARDWARE_MISMATCH_CODE = "baseline_hardware_mismatch"`,
     `BASELINE_TIMING_POLICY_MISMATCH_CODE = "baseline_timing_policy_mismatch"`.
   - A frozen dataclass `CurrentRunEnvironment` with `hardware`, `rocm_version`,
     `target_id`, `timing_policy`.
   - A frozen dataclass `BaselineCoverageEntry` with `workload_key`, `state`,
     `reason_code`, `detail` (str), and optional `sub_codes` (tuple).
   - A frozen dataclass `BaselineCoverageReport` with `entries` (tuple),
     `expected_workload_keys` (tuple), and a `state_summary` property returning
     per-state counts (dict[str, int]); plus `to_dict()`.
   - `validate_baseline_coverage(registry, *, current_run_environment,
     trace_root=None) -> BaselineCoverageReport`. Classification per expected
     `workload_key`:
       1. No matching entry → `missing` / `missing_baseline`.
       2. Entry whose `source`/provenance indicates reference-latency derivation
          (mirrors `_PLACEHOLDER_BASELINE_SOURCES`: `source` references
          `reference_latency` or entry carries a reference/placeholder marker)
          → `placeholder` / `placeholder_baseline`.
       3. `trace_root` provided AND the entry's `trace_ref` file does not exist
          under it → `stale` / `baseline_stale` + `baseline_stale_trace`.
       4. Entry provenance mismatches `current_run_environment`:
          `hardware` differs → `mismatched` / `baseline_mismatched` +
          `baseline_hardware_mismatch`; `timing_policy` differs →
          `baseline_mismatched` + `baseline_timing_policy_mismatch`. (Also
          compare `rocm_version` and `target_id`, surfacing the first
          mismatching field in `detail`.)
       5. Otherwise → `confirmed` (status only, no blocker code).
3. Export the public symbols from `src/sol_execbench/core/evidence/__init__.py`.
4. Create `tests/sol_execbench/core/evidence/test_baseline_coverage.py` with the
   SIX required cases (success criterion #4), each fixture-backed (no GPU):
   - **complete coverage** → every expected key `confirmed`;
     `state_summary["confirmed"] == len(expected)`.
   - **partial coverage** → one expected key missing → `missing` /
     `missing_baseline`; suite summary counts it.
   - **hardware mismatch** → entry provenance `hardware` differs from
     `current_run_environment` → `mismatched` + `baseline_hardware_mismatch`.
   - **timing-policy mismatch** → entry `timing_policy` differs → `mismatched` +
     `baseline_timing_policy_mismatch`.
   - **stale trace pointer** → `trace_ref` file absent under `trace_root` →
     `stale` + `baseline_stale_trace`.
   - **placeholder baseline rejection** → reference-latency-derived entry →
     `placeholder` / `placeholder_baseline`.
   - Add a `to_dict()` shape test and a `confirmed`-is-status-not-blocker test
     (no blocker code emitted for confirmed entries).
