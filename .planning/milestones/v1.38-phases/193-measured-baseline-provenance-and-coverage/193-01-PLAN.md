# Phase 193-01 Plan: Measured Baseline Generation Timestamp (BASE-01)

**Requirement:** BASE-01
**Wave:** 1 (independent — no dependency on other 193 plans)
**Files modified:** `src/sol_execbench/core/evidence/baseline_export.py`, `tests/sol_execbench/core/evidence/test_baseline_export.py`

## Objective

Close the only BASE-01 gap on the pre-existing measured baseline registry:
add a generation timestamp so measured baseline provenance records *when* the
registry was produced. Reuse the existing `baseline_export.py` shape — do NOT
re-model provenance fields (the rest of BASE-01 is already covered).

## Decisions Covered

- D-14: Add a top-level `generated_at` ISO-8601 UTC timestamp to the
  `export_hip_baseline_registry()` registry output.
- D-15: Reuse the existing v1.36/v1.37 freshness-identity timestamp helper
  (`utc_timestamp` from `sol_execbench.core.reports.trust_summary`) using the
  established `generated_at or utc_timestamp()` injection pattern.
- D-16: Keep the schema version at `baseline_registry.v1` /
  `sol_execbench.measured_baseline_registry.v1` — adding `generated_at` is a
  backward-compatible additive field (project convention from 191 D-07).
- D-17: `generated_at` is the input the BASE-02 `stale` detection consumes
  (feeds 193-02); it is not decorative.

## Implementation Steps

1. Read `src/sol_execbench/core/evidence/baseline_export.py` and
   `src/sol_execbench/core/bench/profile_summary/builder.py` (reference for the
   `utc_timestamp` injection pattern) before editing.
2. In `baseline_export.py`:
   - Add `from sol_execbench.core.reports.trust_summary import utc_timestamp`.
   - Add a keyword-only parameter `generated_at: str | None = None` to
     `export_hip_baseline_registry()`.
   - In the returned `registry` dict, add a top-level field
     `"generated_at": generated_at or utc_timestamp()` (placed alongside
     `schema_version` / `sol_schema_version` / `target_id`).
3. Update `tests/sol_execbench/core/evidence/test_baseline_export.py`:
   - Assert the registry payload contains a top-level `generated_at` string
     matching ISO-8601 UTC (ends with `Z`, parseable).
   - Add a test that an explicitly passed `generated_at` value is preserved
     verbatim (injection override), mirroring the profile_summary/agent_feedback
     freshness tests.
   - Do not weaken the existing coverage / NaN / malformed-trace tests.

## must_haves

- D-14: `export_hip_baseline_registry()` accepts a `generated_at` keyword
  argument and writes it as a top-level registry field.
- D-15: The default `generated_at` uses the shared `utc_timestamp()` helper
  (reused from v1.36/v1.37 freshness identity), not a new helper.
- D-14: The written registry JSON contains a top-level `generated_at` ISO-8601
  UTC string when no override is supplied.
- D-14: An explicitly supplied `generated_at` is preserved unchanged.
- D-16: The schema version remains `baseline_registry.v1` /
  `sol_execbench.measured_baseline_registry.v1` (no version bump — additive
  field).
- D-17: No provenance fields are re-modelled; the existing entry shape is
  unchanged (the timestamp feeds 193-02 stale detection).
