# Requirements: SOL ExecBench ROCm Port v1.14

**Defined:** 2026-05-25
**Milestone:** v1.14 Optional rocprofv3 Profiling Evidence
**Status:** ACTIVE
**Source:** `.planning/notes/2026-05-23-rocm-systems-gpuopen-enhancements.md`

## Goal

Add opt-in `rocprofv3` profiling artifacts as benchmark evidence so anomalous
or hardware-sensitive results can be diagnosed without changing correctness,
timing, scoring, or default execution behavior.

## Compatibility Position

v1.14 is planned as a backward-compatible opt-in profiling milestone.

- Profiling is disabled by default.
- `rocprofv3` availability is never required for normal benchmark execution.
- Profiler failure records evidence status but does not convert a correctness
  pass into a correctness failure.
- Existing v1.12/v1.13 contract consumers remain compatible unless they
  explicitly require a new optional profiling capability.

## Requirements

### Profiling Invocation

- [x] **PROF-01**: Add an explicit profiling option such as
  `--profile rocprofv3` for benchmark execution.
- [x] **PROF-02**: Build profiler commands with recorded provenance, working
  directory, environment-relevant arguments, timeout behavior, and output
  locations.
- [x] **PROF-03**: Keep profiling outside default benchmark execution and make
  profiler availability checks clear before profile collection begins.

### Artifact Capture

- [x] **ART-01**: Persist `rocprofv3` artifacts with structured paths,
  including preferred `rocpd` database output when available.
- [x] **ART-02**: Support CSV artifact registration or conversion metadata for
  kernel traces, counter collections, and agent information when configured.
- [x] **ART-03**: Record profiler stdout/stderr tails, exit status, skipped
  reason, and failed reason in result metadata.

### Reporting and Docs

- [x] **REPORT-01**: Report profiling evidence as optional diagnostics, not as
  correctness or score authority.
- [x] **REPORT-02**: Document how to enable profiling in local and Docker ROCm
  environments, including expected privileges and common failure modes.
- [x] **REPORT-03**: Add tests proving profiling unavailable/failed/skipped
  states remain non-fatal for correctness evaluation.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Making profiling mandatory for benchmark runs | Would break ordinary local and CI workflows. |
| ROCm Compute Profiler roofline analysis | Candidate later CDNA-focused analysis work. |
| RGA/code-object/static ISA analysis | Separate static kernel evidence milestone. |
| Contract version bump | Only needed if profiling becomes a required consumer contract. |
