# Phase 122 Plan: Public Publishing Materials

## Objective

Create public publishing materials that let maintainers prepare a GitHub prerelease or equivalent public release page with correct artifact links, support boundaries, research-preview wording, and known limitations.

## Scope

- Add `docs/user/public_prerelease.md`.
- Add `docs/releases/v1_26_prerelease_draft.md`.
- Update README documentation links.
- Add focused docs tests.
- Update milestone state after verification.

## Tasks

1. Write public publishing guide.
   - Checklist from bundle generation through readiness gate and artifact attachment.
   - Required public links.
   - Claim-boundary review checklist.

2. Write release draft.
   - Title, summary, artifacts, validation, research preview, support boundaries, known limitations, and links.
   - Placeholder URLs for attached bundle/readiness assets.
   - Explicit engineering prerelease and research preview wording.

3. Add tests.
   - Ensure release draft and guide contain required links.
   - Ensure wording says engineering prerelease and research preview.
   - Ensure forbidden stable-authority/paper/leaderboard claims do not appear as positive claims.

4. Verify.
   - Run focused publishing docs tests.
   - Run related release/research/docs guardrails.

## Acceptance Checks

- PUBLISH-01: Maintainer can prepare a GitHub prerelease draft or equivalent release page from repo docs.
- PUBLISH-02: Public materials link artifact bundle, support matrix, claim boundaries, first-run guide, timing semantics, researcher guide, and known limitations.
- PUBLISH-03: Public wording describes the result as an engineering prerelease and research preview, not a stable benchmark authority release.
