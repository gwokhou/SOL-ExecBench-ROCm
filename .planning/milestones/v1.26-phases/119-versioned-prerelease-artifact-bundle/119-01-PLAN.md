# Phase 119 Plan: Versioned Prerelease Artifact Bundle

## Objective

Create a maintainable prerelease artifact bundle workflow that emits reviewable JSON/Markdown bundle manifests, checksums, command transcripts, environment evidence references, validation summaries, and authority-class mappings.

## Scope

- Add a CPU-safe bundle generator script under `scripts/`.
- Reuse `scripts/release_candidate_validation.py` as the default validation producer.
- Add focused tests for manifest shape, checksums, authority classes, transcript redaction, optional evidence status, and failure handling.
- Document how maintainers generate and inspect the bundle.
- Update GSD requirement and phase status after verification.

## Tasks

1. Implement `scripts/build_prerelease_artifact_bundle.py`.
   - Accept `--version` and `--output-dir`.
   - Run or skip release validation with recorded transcripts.
   - Optionally record environment evidence through a configurable command.
   - Write `prerelease_artifact_bundle.json`, `prerelease_artifact_bundle.md`, and `SHA256SUMS`.
   - Classify every artifact as `canonical`, `diagnostic-only`, `provisional`, `deferred`, or `unavailable`.

2. Add tests.
   - Verify a successful bundle contains the manifest, checksums, validation outputs, transcripts, claim boundary, and authority classes.
   - Verify missing optional evidence is represented as `unavailable` or `deferred`.
   - Verify failed release validation is represented as blocking/review-needed and transcript tails are redacted.

3. Add documentation.
   - Document the maintainer command and output layout.
   - Document authority classes and claim boundaries.
   - Point reviewers to validation summaries and checksums.

4. Verify.
   - Run focused pytest coverage for the new bundle script.
   - Run the existing release validation tests.
   - Run lint on touched Python files if available.

## Acceptance Checks

- ARTIFACT-01: Maintainer can generate a versioned bundle from a checkout or tag.
- ARTIFACT-02: Bundle exposes checksums, transcripts, environment evidence, and validation summaries.
- ARTIFACT-03: Bundle manifest maps artifacts to the required authority classes.

## Risks

- Running full validation by default may be slow; keep defaults CPU-safe and mockable.
- Optional hardware evidence may be unavailable on most development hosts; represent that explicitly instead of failing bundle creation.
- Bundle evidence must not overclaim release readiness before Phase 120 gates exist.
