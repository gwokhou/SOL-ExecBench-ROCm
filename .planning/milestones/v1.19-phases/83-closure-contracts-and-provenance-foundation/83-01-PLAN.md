<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dataset sidecar JSON -> closure helper models | Untrusted local JSON metadata is validated before report construction. |
| runner/script paths -> sidecar refs | File paths can leak absolute user paths unless callers pass relative refs and helpers avoid expanding payloads. |
| existing trace evidence -> reusable closure status | Stale traces must not be treated as reusable when provenance differs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-83-01 | Tampering | `ExecutionClosureReport` input records | mitigate | Validate statuses/reasons through enums/Pydantic and compute deterministic checksum over model payload. |
| T-83-02 | Repudiation | provenance mismatch reporting | mitigate | Emit stable mismatch diagnostics for manifest/readiness/ready-subset/workload/solution/evidence requirements. |
| T-83-03 | Information Disclosure | source refs and sidecar fields | mitigate | Store refs/checksums only; do not embed raw traces, raw dataset payloads, proprietary kernels, credentials, or unbounded logs. |
| T-83-04 | Elevation of Privilege | claim boundaries | mitigate | Keep paper parity, leaderboard authority, full validation, and score authority false in the sidecar claim boundary. |
| T-83-SC | Tampering | package installs | accept | No package-manager install tasks are planned for this phase. |
</threat_model>
