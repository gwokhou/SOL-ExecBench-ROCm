<interfaces>
Executor should preserve these existing runner-side contracts:

```python
def run_cli(...) -> list[dict] | None
def _save_cli_log(output_dir: Path, job_name: str, result: subprocess.CompletedProcess)
def _closure_record(..., cli_log_ref: str | None = None, evidence_refs: dict[str, str] | None = None, evidence_gaps: list[str] | None = None, trace_status: str | None = None, notes: list[str] | None = None) -> dict
def _derived_evidence_for_workload(...) -> tuple[dict[str, str], list[str]]
def _write_execution_closure(*, path: Path, records: list[dict], provenance: dict, filters: dict) -> None
```

Existing status semantics:
- `attempted_failed` covers attempted workloads with failed traces or runner failure.
- `missing_trace` exists for selected workloads where no trace matches the requested workload.
- `derived_evidence_missing` exists when requested sidecar evidence is absent.
- `filtered` covers max-workloads and missing selected workload rows.
- `not_attempted` covers readiness-blocked workloads after the execution loop.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Subprocess result -> runner closure | CLI stdout, stderr, return code, and timeout behavior are untrusted diagnostic inputs. |
| Trace payload -> closure sidecar | Trace JSON may omit selected workloads or contain failed statuses. |
| Sidecar paths -> report JSON | Local paths must be bounded and relative where possible. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86-05 | Information Disclosure | failed CLI diagnostics | mitigate | Write raw stdout/stderr to per-problem log files and store only bounded relative `cli_log_ref` plus concise notes in closure JSON. |
| T-86-06 | Tampering | trace-to-workload matching | mitigate | Classify selected workloads missing from returned traces as `missing_trace` instead of silently treating the problem summary as authoritative for every selected workload. |
| T-86-07 | Repudiation | failure reason taxonomy | mitigate | Keep stable reason/status strings covered by contract tests and public sidecar-only guardrails. |
| T-86-08 | Denial of Service | unbounded logs in JSON | mitigate | Assert closure JSON omits raw stdout/stderr bodies and absolute temp paths; keep log content in files, not sidecar records. |
| T-86-SC | Tampering | package installs | accept | No package-manager install tasks in this plan. |
</threat_model>
