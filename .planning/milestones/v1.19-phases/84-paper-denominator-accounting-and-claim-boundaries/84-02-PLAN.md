## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> script | User-provided local JSON paths and artifact path refs enter the report workflow. |
| sidecar script -> public CLI expectations | A new utility script must not become primary benchmark CLI authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84-05 | Spoofing | script source refs | mitigate | Preserve path/ref/checksum metadata in the report and rely on core checksum extraction or file sha256 for optional artifact refs. |
| T-84-06 | Tampering | script wrapper | mitigate | Keep wrapper thin and delegate normalization/checksum/rendering to Plan 84-01 core helpers. |
| T-84-07 | Information Disclosure | optional artifact refs | mitigate | Store bounded refs/checksums only; never read or copy AMD SOL/SOLAR artifact payloads into the denominator report. |
| T-84-08 | Elevation of Privilege | primary CLI surface | mitigate | Add guardrails proving the primary CLI does not expose denominator reporting options or authority claims. |
</threat_model>
