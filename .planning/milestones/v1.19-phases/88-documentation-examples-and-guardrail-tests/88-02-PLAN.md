## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fixture examples -> researcher interpretation | Demo fixtures can be mistaken for real benchmark validation or score authority. |
| fixture paths/logs -> public repository | Example artifacts can accidentally expose absolute local paths, secrets, raw data, or unbounded logs. |
| example fields -> public contracts | Sidecar/demo fields can drift into canonical contracts through documentation pressure. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

</threat_model>
