# Roadmap: v1.13 ROCm Runtime Evidence and Environment Diagnostics

**Defined:** 2026-05-25
**Status:** PLANNING
**Source:** `.planning/notes/2026-05-23-rocm-systems-gpuopen-enhancements.md`

## Milestone Goal

Attach trustworthy runtime environment evidence to benchmark runs and provide
fast ROCm diagnostic commands, while preserving v1.12-compatible benchmark
behavior and consumer contracts.

## Phase Structure

### Phase 58: Environment Snapshot Contract

Goal: define the internal data contract and collection boundary for runtime
environment evidence.

Scope:

- Add environment snapshot models and serialization helpers.
- Represent successful, unavailable, skipped, and failed collection outcomes.
- Add bounded subprocess helpers for ROCm tool probes.
- Expose optional `runtime.evidence.v1` capability without bumping contract
  version.

Acceptance:

- Old fixtures parse without environment metadata.
- New environment evidence round-trips through tests.
- Missing ROCm tools produce structured unavailable statuses, not crashes.

### Phase 59: Benchmark Run Evidence Integration

Goal: attach environment snapshots to benchmark execution artifacts without
changing correctness or scoring semantics.

Scope:

- Collect snapshots at run boundaries outside the measured timing window.
- Persist snapshot metadata in optional result/run metadata.
- Ensure canonical trace fields and public schemas remain stable.
- Add guardrail tests for v1.12 compatibility and no scoring dependency.

Acceptance:

- Non-profiled benchmark output can include environment evidence when
  collection is enabled or available.
- Absence of environment evidence does not change benchmark pass/fail or score.
- Contract and trace guardrails remain compatible with hip-playground-style
  consumers.

### Phase 60: Diagnostics CLI and Preflight Checks

Goal: give users a standalone way to diagnose ROCm, Docker, and GPU runtime
readiness before running a full benchmark.

Scope:

- Add `sol-execbench doctor` or `sol-execbench env-snapshot`.
- Emit JSON output for tool availability and runtime readiness.
- Add lightweight HIP/ROCm smoke checks with architecture-aware skip behavior.
- Document local and Docker usage.

Acceptance:

- Diagnostic command runs without a problem directory or solution.
- CPU-only/default test environments skip hardware-only checks cleanly.
- Documentation lets a user distinguish SOL ExecBench failures from ROCm
  installation or device visibility failures.

## Verification Plan

- Unit tests for environment snapshot model parsing and serialization.
- Unit tests for unavailable/error probe handling.
- CLI tests for diagnostic command JSON output.
- Contract guardrails proving contract version remains `1.0`.
- Public trace/schema guardrails proving optional evidence does not affect
  canonical correctness, timing, or scoring fields.

## Compatibility Notes

This milestone is intentionally non-breaking. Any future decision to require
environment evidence for consumer-side `confirmed_benchmark` claims belongs in
a separate explicit contract-bump milestone or consumer integration milestone.
