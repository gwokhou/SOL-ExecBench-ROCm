# Phase 135 Plan: Dataset Runner Integration and Public Guardrails

**Created:** 2026-06-04
**Status:** Ready for execution

## Goal

Integrate migrated dataset manifests, readiness sidecars, and ready subsets
into `scripts/run_dataset.py` closure outputs and public documentation while
preserving local-only dataset guardrails and existing idempotent runner
behavior.

## Work Items

1. Runner metadata and closure enrichment
   - Add package helper logic for extracting migration manifest metadata,
     license boundary, readiness/ready-subset denominator summaries, claim
     boundaries, and blocker summaries.
   - Include that metadata in execution closure provenance and source refs.
   - Include readiness class, blocker types/codes, and blocker evidence refs in
     per-workload closure rows without breaking existing fields.

2. Idempotent ready-subset behavior
   - Preserve trace reuse only when manifest/readiness/ready-subset checksums,
     solution mode, runtime config, and requested evidence match.
   - Ensure closure-only runs with no selected runnable workloads write stable
     summary and closure reports instead of silently dropping blockers.

3. Tests
   - Add CPU-safe tests for migrated manifest closure metadata, public-safe
     license-boundary provenance, readiness filtering/blocker denominator
     records, idempotent summary/closure behavior, and reuse invalidation for
     provenance drift.
   - Add docs/guardrail tests if existing public-doc tests do not already cover
     the new wording.

4. Documentation and release guardrails
   - Update user-facing docs with local download/migration, readiness subset,
     and bounded runner commands.
   - State that NVIDIA/SOL-ExecBench source or migrated derivative content is
     not redistributed and that low-precision/CDNA3/CDNA4 evidence remains
     unvalidated unless a complete hardware evidence chain exists.

## Risks

- Execution closure schema has `extra="forbid"`, so added fields must be
  explicit and tested.
- Existing runner tests likely assert exact provenance keys; changes must be
  additive and deterministic.
- Public docs must avoid implying redistributed NVIDIA dataset content is
  available from this repository.
