## Integration / Coherence Checks

1. **Confirmed-evidence chain is end-to-end coherent:**
   - 190 registers profiler artifacts + citations (diagnostic).
   - 191 structures profile summary evidence (diagnostic).
   - 192 adds the official score gate + blocker vocabulary (STAGING).
   - 193 adds measured baseline provenance (`generated_at`), five-state coverage
     validation, and wires coverage as the gate's precondition (STAGING
     preserved).
   - 194 advertises confirmed capabilities on the contract, ships the
     `sol-execbench official-score` emission CLI, and publishes HIP-facing
     fixtures + docs. Valid runs remove `missing_score`/`missing_baseline`/
     `placeholder_baseline`; invalid runs keep precise codes.
2. **Authority boundaries preserved:** only `official_score` and
   `measured_baseline` are `confirmed`; all sidecars (agent_feedback,
   profile_summary, environment_budget, static_resource_footprint, decision)
   remain `diagnostic`. `reference_latency_ms` stays blocked for confirmed
   claims.
3. **Schema stability:** contract stays `sol_execbench.evaluator_contract.v2`
   (additive); measured baseline registry stays `baseline_registry.v1`;
   `official_score_evidence.v1` introduced. The contract capability table and
   builder are kept in sync by `test_current_contract_doc_matches_builder_capabilities`.
4. **Blocker vocabulary consistency:** the contract's `confirmed_evidence_blockers`
   is pinned equal to `official_score`'s blocker constants; the coverage module's
   `missing_baseline`/`placeholder_baseline` literals are pinned equal to the
   gate's constants (no parallel namespace drift).
5. **CPU-safe:** all new code is GPU-free and fixture-backed; no real GPU
   required for any new test.
