# v1.31 Closure Handoff

## Status

v1.31 RDNA4 Follow-Up Evidence Hardening is complete from a milestone
requirements perspective after Phases 142-147. Phase 147 first attempted
memory-pressure reduction and timing fallback root-cause analysis before
recording residual boundaries.

No claim upgrade is made by this handoff. The supported claim remains bounded
RDNA4 `gfx1200` ready-subset evidence for the recorded host, commands,
exclusions, and artifacts only.

## Completed In This Milestone

- Clock sudoers coverage and live RDNA4 clock-lock/reset evidence were captured.
- RDNA4 timing was rerun and explicitly classified as PyTorch/device-event
  fallback rather than profiler-backed timing; Phase 147 confirmed this is
  caused by PyTorch source-language policy routing, not missing `rocprofv3`.
- Phase 140 derived sidecar exclusions were retried through isolated
  memory-capped jobs.
- All 12 `missing_trace` rows were classified as `gpu_oom_no_trace`.
- All 146 failed workload records were classified into seven failure classes.
- Phase 147 reduced avoidable eval-driver memory retention by avoiding
  unconditional reference-output clones and releasing correctness outputs
  before timing.
- Phase 147 reran a known RDNA4 memory-failed workload and confirmed the
  sampled residual blocker is inside the reference/user function peak VRAM
  requirement on the 16 GiB validation GPU.
- Public docs were updated to cite v1.30/v1.31 bounded evidence without
  upgrading RDNA4 validation authority.
